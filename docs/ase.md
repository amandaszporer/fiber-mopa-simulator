This is the single source of truth for Amplified Spontaneous Emission (ASE)
modeling in the simulator: the physics, equations, algorithms, data structures,
and validation, plus a plain-language walkthrough of how the solver actually
works.

It is organized in two parts:

- **Part I ‚Äî Specification.** The rigorous reference: every equation, the

- **Part II ‚Äî Walkthrough.** A step-by-step, no-programming-required explanation
  of how the spectrally-resolved solver computes ASE inside each amplifier

The ASE solver lives in the `ase/` package; this document is its companion.

# Part I ‚Äî Specification

## 1. What ASE Is and Why It Matters

Amplified Spontaneous Emission is light generated by spontaneous decay of
excited Yb¬≥‚Å∫ ions that then gets amplified as it propagates through the inverted
fiber. Unlike the signal (a coherent pulse at 1064 nm), ASE is broadband
(~970‚Äì1130 nm), incoherent, propagates in both directions, and acts as parasitic
noise that:

1. **Depletes the inversion** ‚Äî ASE consumes stored energy that should go to the
   signal pulse.
2. **Limits achievable gain** ‚Äî at high enough gain, ASE clamps the inversion
   (ASE-limited gain).
3. **Can cause parasitic lasing** ‚Äî if end-facet reflections create a round-trip
   gain > 1 at any wavelength.
4. **Degrades pulse energy at low rep rates** ‚Äî between pulses, ASE builds up and
   eats stored energy.
5. **Adds noise to the output** ‚Äî forward ASE co-propagates with the signal.

A simulator that estimates ASE as a single number (like
`P_ASE ‚âà n_sp ¬∑ (G-1) ¬∑ hŒΩ ¬∑ ŒîŒΩ`) misses all of the above dynamics. Accurate ASE
modeling requires spectrally resolved, bidirectional propagation coupled to the
time-dependent rate equations.

## 2. The Equation Set

### 2.1 Variables

At every position z along the fiber and time t:

```
n‚ÇÇ(z, t)    ‚Äî fraction of Yb ions in the excited state (0 ‚â§ n‚ÇÇ ‚â§ 1)
N            ‚Äî total Yb¬≥‚Å∫ ion density [ions/m¬≥] (constant, derived from cladding absorption)
N‚ÇÇ = n‚ÇÇ ¬∑ N  ‚Äî excited-state population density
N‚ÇÅ = (1 - n‚ÇÇ) ¬∑ N  ‚Äî ground-state population density

P_pump(z, t)       ‚Äî pump power [W] (forward or backward)
P_signal(z, t)     ‚Äî signal power [W] (forward)
P_ASE_fwd(z, t, Œª) ‚Äî forward ASE power in wavelength bin centered at Œª [W]
P_ASE_bwd(z, t, Œª) ‚Äî backward ASE power in wavelength bin centered at Œª [W]
```

### 2.2 Propagation equations

For each optical channel k with direction u_k = +1 (forward) or -1 (backward):

```
(1/v_g) ‚àÇP_k/‚àÇt + u_k ¬∑ ‚àÇP_k/‚àÇz = u_k ¬∑ g_k(z,t) ¬∑ P_k(z,t)
                                    + u_k ¬∑ S_k(z,t)
                                    - u_k ¬∑ Œ±_bg ¬∑ P_k(z,t)
```

Where:
- `v_g = c / n_glass` ‚Äî group velocity in fiber (~2.07 √ó 10‚Å∏ m/s for n_glass = 1.45)
- `g_k(z,t)` ‚Äî gain/absorption coefficient for channel k
- `S_k(z,t)` ‚Äî spontaneous emission source (only nonzero for ASE channels)
- `Œ±_bg` ‚Äî background fiber loss (~0.005 dB/m ‚âà 0.00115 Np/m at 1064 nm)

### 2.3 Gain coefficient

```
g_k(z,t) = Œì_k ¬∑ N ¬∑ [n‚ÇÇ(z,t) ¬∑ œÉ_e(Œª_k) - (1 - n‚ÇÇ(z,t)) ¬∑ œÉ_a(Œª_k)]
```

Where:
- `Œì_k` ‚Äî overlap factor between mode at wavelength Œª_k and doped core
- `œÉ_e(Œª_k)` ‚Äî emission cross-section at wavelength Œª_k [m¬≤]
- `œÉ_a(Œª_k)` ‚Äî absorption cross-section at wavelength Œª_k [m¬≤]

For the **pump** channel in a double-clad fiber:
```
Œì_pump = A_core / A_clad
```

For the **signal** and **ASE** channels (core-guided):
```
Œì_k = 1 - exp(-2 ¬∑ (r_core / w(Œª_k))¬≤)
```
where `w(Œª_k)` is the mode radius at that wavelength, computed from the Marcuse
approximation with wavelength-dependent V-number:
```
V(Œª) = (2œÄ / Œª) ¬∑ r_core ¬∑ NA
w(Œª) / r_core = 0.65 + 1.619 / V(Œª)^1.5 + 2.879 / V(Œª)^6    (if V < 2.405)
w(Œª) = 0.65 ¬∑ r_core                                           (if V > 2.405, fundamental mode)
```

**Implementation note**: Pre-compute `Œì_k`, `œÉ_e(Œª_k)`, `œÉ_a(Œª_k)` for every ASE
bin once at initialization. Store as arrays indexed by bin number. These do not
change during simulation.

### 2.4 Spontaneous emission source term

```
S_k(z,t) = m_pol ¬∑ h ¬∑ ŒΩ_k ¬∑ ŒîŒΩ_k ¬∑ Œì_k ¬∑ œÉ_e(Œª_k) ¬∑ N ¬∑ n‚ÇÇ(z,t)
```

Where:
- `m_pol = 1` for PM (polarization-maintaining) fiber
- `m_pol = 2` for non-PM fiber (two orthogonal polarization modes)
- `h = 6.626 √ó 10‚Åª¬≥‚Å¥` J¬∑s
- `ŒΩ_k = c / Œª_k` ‚Äî frequency of ASE bin k [Hz]
- `ŒîŒΩ_k` ‚Äî width of ASE bin k in Hz: `ŒîŒΩ_k = c ¬∑ ŒîŒª / Œª_k¬≤`

This term is the seed that starts ASE. It represents the fraction of
spontaneously emitted photons that couple into the guided mode. One source term
per ASE bin, per direction (forward and backward get identical source terms ‚Äî
spontaneous emission is isotropic in the guided mode).

**The source term is NOT applied to the pump or signal channels** ‚Äî only to ASE bins.

### 2.5 Rate equation for upper-state population

```
‚àÇn‚ÇÇ/‚àÇt = R_abs(z,t) - R_em(z,t) - n‚ÇÇ/œÑ
```

Where the absorption and emission rates sum over ALL channels (pump + signal +
every ASE bin in both directions):

```
R_abs = Œ£_k [ œÉ_a(Œª_k) ¬∑ Œì_k ¬∑ P_k(z,t) / (h ¬∑ ŒΩ_k ¬∑ A_k) ] ¬∑ (1 - n‚ÇÇ)

R_em  = Œ£_k [ œÉ_e(Œª_k) ¬∑ Œì_k ¬∑ P_k(z,t) / (h ¬∑ ŒΩ_k ¬∑ A_k) ] ¬∑ n‚ÇÇ
```

Where `A_k` is the area the dopant ions occupy. **The implementation uses
`A_k = A_core` for ALL channels, including the pump** (with Œì_pump = A_core/A_clad
expressing the cladding-mode overlap). This is the textbook convention (Becker
et al.; Snitzer & Po) and avoids double-counting the pump's geometric dilution ‚Äî
see Part II ¬ß6 for the derivation. The lifetime term `- n‚ÇÇ/œÑ` is spontaneous
decay (œÑ = 0.83 ms for Yb, the measured Melkumov AS value).

**Steady-state form** (set ‚àÇn‚ÇÇ/‚àÇt = 0, solve for n‚ÇÇ):

```
n‚ÇÇ(z) = [Œ£_k œÉ_a(Œª_k) ¬∑ Œì_k ¬∑ P_k / (h¬∑ŒΩ_k¬∑A_core)]
       / [Œ£_k (œÉ_a(Œª_k) + œÉ_e(Œª_k)) ¬∑ Œì_k ¬∑ P_k / (h¬∑ŒΩ_k¬∑A_core) + 1/œÑ]
```

This is used in the inter-pulse BVP solver and for CW analysis.

## 3. Spectral Discretization

### 3.1 Wavelength grid

```
Œª_min  = 970 nm     (must include the 975 nm absorption/emission peak)
Œª_max  = 1130 nm    (tail of Yb emission)
ŒîŒª     = 1 nm       (standard resolution)
N_bins = (Œª_max - Œª_min) / ŒîŒª = 160 bins
```

Each bin has a center wavelength `Œª_i = Œª_min + (i + 0.5) ¬∑ ŒîŒª` and frequency
width `ŒîŒΩ_i = c ¬∑ ŒîŒª / Œª_i¬≤`.

**The signal channel at 1064 nm is a separate channel, not an ASE bin.** It has
its own power variable with different initial conditions and no spontaneous
emission source. The pump at 976 nm is a separate channel too.

Total optical channels: 1 pump + 1 signal + 160 forward ASE + 160 backward
ASE = **322 channels**.

> **Grid note (915 nm pumping).** The pump/signal scalar channels resolve any
> wavelength inside the cross-section data range (848‚Äì1180 nm), so 915 nm
> pumping works. The ASE *bin* grid default stays at [970, 1130] nm ‚Äî widening
> `lambda_min` changes `n_bins` and every ASE result, so it is left as an
> explicit caller choice in `spectral_grid.from_fiber`.

### 3.2 Cross-section spectrum data

`œÉ_a(Œª)` and `œÉ_e(Œª)` are needed as arrays sampled at each bin center. The
simulator ships the **measured aluminosilicate (AS) cross-section table** of
Melkumov et al. in `ase/data/cross_sections_yb.csv` ‚Äî the closest published
host-glass match for the Coherent/Nufern SM-YDF-5/130-VIII fiber. The table is
the measured source of truth; it is *not* generated from anchors.

> **Provenance.** The full numeric table comes from the **non-peer-reviewed
> preprint** (M.A. Melkumov, I.A. Bufetov, K.S. Kravtsov, A.V. Shubin,
> E.M. Dianov, *"Absorption and emission cross section of Yb3+ ions in Al2O3
> and P2O5 doped fibers,"* FORC Preprint No. 5, Moscow, 2004; arXiv:1502.02885,
> Appendix 2, AS columns). A **peer-reviewed** English/Russian journal version
> of the same study exists (Mel'kumov et al., *"Lasing parameters of
> ytterbium-doped fibres doped with P2O5 and Al2O3,"* Quantum Electronics
> 34(9), 843‚Äì848 (2004), DOI 10.1070/QE2004v034n09ABEH002688) but is a shorter
> presentation that does **not** reproduce the Appendix-2 table ‚Äî so the table
> itself is not peer-reviewed, though a peer-reviewed version of the work
> exists. Published units are pm¬≤ (= 10‚Åª¬≤‚Å¥ m¬≤); the CSV stores m¬≤ (√ó 1e-24).
> Lifetime œÑ = 0.83 ms (AS). The raw published-pm¬≤ transcription is kept at
> `ase/data/melkumov_AS_raw.csv` for provenance.

Representative measured AS values (from the CSV; the absolute scale cancels under
the N back-derivation in ¬ß4.4 of `physics_spec.md`, so only the spectral *shape*
matters physically):

```
Œª [nm]    œÉ_a [10‚Åª¬≤‚Åµ m¬≤]    œÉ_e [10‚Åª¬≤‚Åµ m¬≤]
  915        6.5                 0.22
  940        3.2                 0.53
  976       26.9                29.7          ‚Üê zero line (œÉ_e/œÉ_a ‚âà 1.10)
  1000       0.99                3.6
  1010       0.86                4.9
  1020       0.70                6.5
  1030       0.43                6.2          ‚Üê emission shoulder peak (>1020 nm)
  1064       0.046               3.0          ‚Üê signal wavelength (œÉ_e/œÉ_a ‚âà 65)
  1080       0.018               2.3
  1100       0.0056              1.6
  1130       ~0.0001             ~0.06
```

Note œÉ_e(1030) > œÉ_e(1010) > œÉ_e(1000): the real Yb:silica emission shoulder
peaks just past 1020 nm.

**Interpolation: piecewise log-linear, monotone between measured nodes.** The
data is interpolated onto each fiber's spectral grid at startup; log-linear is
monotone between nodes by construction ‚Üí zero overshoot near the narrow 976 nm
peak (FWHM ‚âà 7.7 nm) and the 1030 nm shoulder. œÉ is clamped to a positive floor
before `log()`. A single shared helper (`cross_sections._log_linear_interp`) is
used by both `cross_sections.py` and `dopants.py` so the two paths cannot
diverge. Out-of-range wavelengths raise `ValueError` (no extrapolation).

**Critical**: the 970‚Äì1000 nm region has LARGE absorption cross-sections. ASE
generated near 975 nm (where œÉ_e is huge) gets strongly reabsorbed near the
pump-input end of the fiber. This spectral reshaping is a key physical effect
that only a spectrally resolved model captures.

Verify convergence by halving ŒîŒª (to 0.5 nm, 320 bins) and confirming that total
integrated ASE power changes by a small amount (the test suite asserts < 5%).

## 4. Solver Architecture

The simulation operates in two modes that hand off to each other.

### 4.1 Mode A: Steady-state BVP solver (inter-pulse equilibrium)

**When to use**: computing the inversion profile and ASE distribution either
(a) at CW equilibrium, or (b) at the end of a long inter-pulse interval when the
system has reached quasi-steady-state.

**What it solves**: the propagation equations with ‚àÇ/‚àÇt = 0:

```
u_k ¬∑ dP_k/dz = u_k ¬∑ g_k(z) ¬∑ P_k(z) + u_k ¬∑ S_k(z) - u_k ¬∑ Œ±_bg ¬∑ P_k(z)
```

plus the steady-state inversion (¬ß2.5).

**Boundary conditions**:

```
At z = 0 (input end):
    P_pump_fwd(0)    = P_pump_input            (known)
    P_signal_fwd(0)  = P_signal_input          (known)
    P_ASE_fwd(0, Œª)  = P_ASE_input_fwd(Œª)      (known ‚Äî zero for first stage, or output of previous stage)
    P_ASE_bwd(0, Œª)  = ???                     (UNKNOWN ‚Äî this is what we solve for)

At z = L (output end):
    P_ASE_bwd(L, Œª)  = R_output ¬∑ P_ASE_fwd(L, Œª)  (reflection; R < 10‚Åª‚Å¥ for angle-cleaved ‚Äî see ¬ß5.3)
    P_pump_bwd(L)    = R_output ¬∑ P_pump_fwd(L)     (if counter-pumped, set separately)
```

This is a two-point BVP because forward channels are known at z=0, backward
channels are known at z=L.

**Algorithm: iterative shooting**

```
1. Initialize: seed P_ASE_bwd(0, Œª) (and P_ASE_fwd) from spontaneous emission
   on the pump-only inversion profile ‚Äî NOT zero. (See note below.)
2. Loop until converged:
   a. Compute n‚ÇÇ(z) at every grid point from current {P_k(z)}
   b. Integrate forward channels from z=0 to z=L using RK4
   c. Apply boundary condition at z=L: P_ASE_bwd(L) = R ¬∑ P_ASE_fwd(L)
   d. Integrate backward channels from z=L to z=0 using RK4
   e. Check convergence: max |P_k_new - P_k_old| / P_k_old < tol for all k
   f. If not converged, update n‚ÇÇ(z) and repeat from (b)
3. Return: n‚ÇÇ(z), all P_k(z) profiles
```

Convergence tolerance: `tol = 1e-5` (relative). Spatial grid: 100‚Äì500 points per
fiber (default 200). See Part II ¬ß10‚Äì¬ß12 for why the **local-n‚ÇÇ update inside
each sweep** (not a frozen-n‚ÇÇ iteration) is what keeps high-gain stages stable.

**Initial-guess note (spontaneous-emission seed).** Step 1 does *not* start the
backward ASE at zero. A constant-power, zero-ASE trial lets a high-gain fiber
amplify a negligible source into a runaway on the first sweep and converge to an
unphysical ASE-dominated fixed point. Instead, a single pump-only forward sweep
(signal = 0, ASE source off) yields an exact pump-only inversion n‚ÇÇ(z); the
spontaneous-emission source evaluated on that profile is accumulated gain-free
from each facet to seed both the forward and backward ASE fields. This
physically-motivated lower-bound floor lands the iteration in the physical
(signal-dominated) basin. It degrades gracefully ‚Äî at zero pump the inversion is
zero, so the seed collapses to the historical zero-ASE guess. Implemented in
`ase/solver_steady.py::_spontaneous_emission_init`; a caller may still override
it via the `init=` parameter (used by the homotopy warm-starts, ¬ß4.4 / Part II
¬ß13b).

**Alternative: scipy.integrate.solve_bvp.** The same problem can be posed as a
relaxation problem and handed to `solve_bvp` (collocation, Kierzenka & Shampine
2001). It is more robust than shooting for high-gain systems near lasing
threshold, at higher cost. The simulator uses iterative shooting with the
local-n‚ÇÇ update; `solve_bvp` is noted as the evaluated-but-not-adopted
alternative.

### 4.2 Mode B: Time-dependent pulse propagation

**When to use**: simulating (a) the ns pulse passing through the fiber, and
(b) the inter-pulse inversion recovery with ASE buildup.

**What it solves**: the full time-dependent equations from ¬ß2.

#### Sub-regime B1: Inter-pulse recovery (¬µs‚Äìms timescale)

Between pulses, there is no signal. The pump rebuilds n‚ÇÇ(z) while ASE grows. The
optical transit time (~5 ns/m) is negligible compared to the evolution
timescale, so the `(1/v_g) ‚àÇP/‚àÇt` term is dropped. **All ASE accounting happens
here.** Operator-split: the spatial P-field is re-solved (one forward + backward
sweep at fixed n‚ÇÇ) only when n‚ÇÇ has drifted by >5% from the last solve; in
between, the n‚ÇÇ rate equation is integrated forward in time with adaptive dt
(10 ns‚Äì10 ¬µs, target ‚â§ 1% Œîn‚ÇÇ per step).

**Simplifications**:
- *Period ‚â´ œÑ (low rep rate)*: skip the time-stepping and run the steady-state
  BVP with signal = 0 to get the fully-recovered inversion. Exact in the limit
  of infinite inter-pulse time.
- *Period ‚â™ œÑ (high rep rate)*: the inversion barely changes between pulses. Run
  the steady-state BVP with the time-averaged signal power as the signal channel
  (the quasi-CW operating point).

#### Sub-regime B2: Pulse propagation (ns timescale)

During the pulse the propagation PDE is solved for **the signal and the pump
only**:

```
(1/v_g) ‚àÇP_k/‚àÇt + ‚àÇP_k/‚àÇz = g_k ¬∑ P_k - Œ±_bg ¬∑ P_k       (k ‚àà {pump, signal})
‚àÇn‚ÇÇ/‚àÇt = R_abs - R_em - n‚ÇÇ/œÑ
```

**ASE is neglected during B2.** ASE accumulates on the ¬µs‚Äìms inter-pulse
timescale (B1); over a 4‚Äì8 ns pulse it has no time to build up meaningfully and
cannot drain the inversion. This is the standard two-part MOPA decomposition:
B1 = pumping + ASE recovery ‚Üí produces the inversion the pulse will see;
B2 = ASE-free pulse extraction ‚Üí produces the amplified pulse energy and the
residual inversion. The two are iterated to periodic steady state.

**Algorithm: Lax-Wendroff finite difference** on a (z, t) grid:
```
Œîz = L / N_z           (N_z = 200 spatial points)
Œît = Œîz / v_g          (CFL = 1 ‚Äî exactly one grid cell per time step)
N_t = pulse_window / Œît (pulse_window ‚âà 3 √ó pulse_duration)
```

At CFL = 1 the upwind step reduces to the method of characteristics with an exact
per-cell exponential transmission `exp((g ‚àí Œ±_bg)¬∑Œîz)` plus an upstream-source
term. At each time step, for the pump and signal channels only:
```
forward (u_k = +1):
    P_k(z_i, t+Œît) = P_k(z_i, t) - (v_g Œît / Œîz) ¬∑ [P_k(z_i, t) - P_k(z_{i-1}, t)]
                     + Œît ¬∑ v_g ¬∑ g_k(z_i) ¬∑ P_k(z_i, t)

Update n‚ÇÇ(z_i):
    n‚ÇÇ(z_i, t+Œît) = n‚ÇÇ(z_i, t) + Œît ¬∑ [R_abs(z_i, t) - R_em(z_i, t) - n‚ÇÇ(z_i, t)/œÑ]
```

**Boundary conditions during pulse**:
```
P_signal_fwd(z=0, t) = input_pulse_shape(t)    (Gaussian)
P_pump_fwd(z=0, t)   = P_pump (CW ‚Äî the pump stays on through the pulse)
```

### 4.3 Handoff between modes

```
Full simulation cycle for one rep-rate period:

1. INTER-PULSE: Start from depleted n‚ÇÇ(z) after previous pulse.
   Run Mode B1 (or Mode A if period >> œÑ) for duration 1/f_rep.
   Result: recovered n‚ÇÇ(z) profile just before pulse arrives.

2. PULSE: Run Mode B2 with n‚ÇÇ(z) from step 1 as initial condition.
   Signal pulse enters at z=0, propagates, extracts energy.
   Result: amplified pulse shape, depleted n‚ÇÇ(z).

3. Repeat for next pulse (or detect convergence to periodic steady state):
   max |n‚ÇÇ_before_pulse(z, cycle N) - n‚ÇÇ_before_pulse(z, cycle N-1)| < cycles_tol

Default cycles_tol = 1e-3, cycles_max = 25.
```

### 4.4 The layered robust steady-state solver

`Amplifier.propagate(mode="steady")` does not call the bare shooter directly ‚Äî it
calls `solve_steady_state_robust`, a three-layer cascade that catches
energy-conservation‚Äìviolating fixed points (a high-gain BVP can admit multiple
self-consistent solutions; see Part II ¬ß13b for the full design, diagnostics, and
literature backing):

1. **Layer 1 ‚Äî direct solve + health diagnostics.** Run Mode A, then check the
   global photon balance against the quantum-defect ceiling
   `signal_gain + ASE_out + bg_loss ‚â§ QD ¬∑ pump_absorbed`. Within ~1% ‚Üí return;
   1‚Äì5% over ‚Üí return with warning; >5% ‚Üí Layer 2.
2. **Layer 2 ‚Äî Ren et al. 2015 homotopy continuation.** Walk from an easy
   operating point (pump-only ‚Üí add signal ‚Üí enable ASE source ‚Üí ramp pump
   50%‚Üí75%‚Üí100%), each step warm-starting the next via the `init=` parameter,
   with an Xu et al. 2014 linear-gain-shape multistart as final fallback.
3. **Layer 3 ‚Äî time-marching arbiter.** Dispatch to `solve_time_dependent(mode="full")`;
   the time-dependent IVP has a unique attractor by construction, so it cannot
   land on an energy-violating fixed point. If even that bails, return Mode A's
   runaway-clamped result with `solver_path_used="all_failed"`.

## 5. Multi-Stage Coupling

### 5.1 Forward ASE propagation between stages

After stage i, the forward ASE spectrum at z = L passes through inter-stage
components, each applying a spectral transfer function:

```
Isolator:        T(Œª) = 10^(-loss_dB / 10)    for all Œª (flat, ~0.5‚Äì1.0 dB)
                 Backward: T = 0 (blocks backward propagation between stages)

Bandpass filter: T(Œª) = T_peak ¬∑ exp(-4¬∑ln2¬∑((Œª - Œª_center)/FWHM)¬≤)
                 Typical: Œª_center = 1064 nm, FWHM = 2 nm, T_peak ‚âà -0.5 dB
                 Crushes the 1030 nm ASE peak before the next gain stage.

MFA / combiner:  T(Œª) = 10^(-loss_dB / 10)    flat loss

Full inter-stage transfer:
P_ASE_fwd_input_next(Œª) = P_ASE_fwd_out(Œª) ¬∑ T_iso(Œª) ¬∑ T_filter(Œª) ¬∑ T_mfa(Œª)
```

### 5.2 Backward isolation

Isolators between stages block backward ASE from stage i+1 reaching stage i, so
each stage's backward ASE is self-contained. In the simulation, **each stage is
solved independently for backward ASE**. With imperfect isolation (e.g. 30 dB),
a small fraction couples back:

```
P_ASE_bwd_input(z=L, stage i) = P_ASE_bwd_output(z=0, stage i+1) ¬∑ 10^(-isolation_dB/10)
```

Negligible at 30 dB, but the model supports it.

### 5.3 End-facet reflections

```
Flat-cleaved:     R ‚âà 3.5% = 0.035
Angle-cleaved 8¬∞: R < 10‚Åª‚Å¥      ‚Üê simulator default
AR-coated:        R < 10‚Åª¬≥
```

BVP boundary conditions:
```
P_ASE_bwd(z=L) = R_output ¬∑ P_ASE_fwd(z=L)      for each ASE bin
P_ASE_fwd(z=0) = R_input ¬∑ P_ASE_bwd(z=0)        for each ASE bin
```

**Parasitic lasing condition**: if the single-pass gain G(Œª) at any wavelength
satisfies
```
G(Œª)¬≤ ¬∑ R_input ¬∑ R_output ‚â• 1
```
the system lases parasitically at that wavelength. The solver checks this after
convergence and flags the stage (it fires only when both reflectivities are
non-zero).

## 8. Algorithm: Complete Simulation Flow

```
INITIALIZATION:
1. Create SpectralGrid (wavelength bins, cross-sections, overlap factors)
2. Create fiber specifications for each stage
3. Define seed pulse (average power, rep rate, pulse duration, linewidth)
4. Define passive components with spectral transfer functions

STAGE-BY-STAGE SIMULATION (for each stage i):

    INPUT: P_signal_in, P_ASE_fwd_in(Œª) (zero for stage 1, else filtered
           output of stage i-1), P_pump for this stage.

    STEP 1: INTER-PULSE STEADY STATE
    - signal = 0 (or time-averaged signal power if rep is high)
    - Solve the steady-state BVP with pump + ASE
    - Result: n‚ÇÇ(z), P_ASE_fwd(z,Œª), P_ASE_bwd(z,Œª); check parasitic lasing

    STEP 2 (if fully time-resolved): INTER-PULSE DYNAMICS (Mode B1)
    - Time-step through the inter-pulse interval from the depleted n‚ÇÇ(z)

    STEP 3: PULSE PROPAGATION (Mode B2, Lax-Wendroff)
    - Propagate signal pulse + pump; n‚ÇÇ(z) depletes; ASE neglected on ns scale

    STEP 4: EXTRACT OUTPUT
    - Signal gain, forward ASE spectrum at z=L, backward ASE spectrum at z=0

    STEP 5: APPLY INTER-STAGE COMPONENTS
    - Multiply P_ASE_fwd(Œª) by T(Œª); apply signal insertion loss; feed next stage

FINAL OUTPUT: amplified pulse, total/forward ASE spectrum, per-stage gain &
ASE power, parasitic-lasing warnings, signal-to-ASE ratio (OSNR).
```

## 9. Simplification Levels

Valid intermediate implementations, most to least simplified. The simulator
implements **Level 5** (and auto-dispatches to the Level-3 steady state when the
rep rate is high enough that the inversion barely changes between pulses).

- **Level 1** ‚Äî steady-state spatial + single ASE band (the old scalar model).
  Can underestimate ASE by 10‚Äì100√ó in high-gain stages.
- **Level 2** ‚Äî steady-state spatial + spectrally resolved *forward* ASE only.
  Misses ~50% of ASE (backward contributes to inversion depletion).
- **Level 3** ‚Äî steady-state BVP with full bidirectional spectral ASE. The
  minimum viable accurate model; valid when rep_rate ‚â´ 1/œÑ (~1.2 kHz).
- **Level 4** ‚Äî Level 3 + pulsed energy extraction (Frantz-Nodvik). Captures
  rep-rate dependence without a full time-domain solver.
- **Level 5** ‚Äî full time-dependent (z, t) solver (¬ß4.2‚Äì¬ß4.3). Captures pulse
  shape distortion and inter-pulse ASE dynamics. Most accurate, most expensive.

## 11. Common Implementation Pitfalls

1. **Forgetting the 975 nm region.** Starting the grid at 1000 nm misses the
   strong backward ASE near 975 nm. Always start at or below 970 nm.
2. **Using A_core for pump intensity geometry incorrectly.** The pump propagates
   in the cladding; the correct rate is `œÉ_a ¬∑ P / (h¬∑ŒΩ¬∑A_clad)`, obtained in the
   implementation via `A_dope = A_core` with `Œì_pump = A_core/A_clad` (Part II ¬ß6).
3. **Not summing ASE in the rate equation.** n‚ÇÇ(z) depends on ALL channels ‚Äî
   pump, signal, and every ASE bin in both directions. Omitting ASE overestimates
   the inversion and the gain.
4. **Sign conventions for backward channels.** Backward channels propagate in -z;
   integrate from z=L to z=0 (or negate the equation and integrate forward).
5. **Numerical instability at high gain.** If small-signal gain exceeds ~30 dB,
   forward propagation can overflow. The local-n‚ÇÇ update (Part II ¬ß11), the
   runaway sentinel, and the layered robust solver address this.
6. **Not checking parasitic lasing.** If round-trip gain exceeds 1, no steady
   state exists; detect before/after the solve.
7. **ŒîŒΩ vs ŒîŒª confusion.** The source term uses ŒîŒΩ (frequency width):
   `ŒîŒΩ_i = c ¬∑ ŒîŒª / Œª_i¬≤`, which is not equal across bins.

# Part II ‚Äî Walkthrough

This part explains, step by step, how the simulator computes ASE inside each
amplifier stage. No programming knowledge is required. It uses the equations
defined rigorously in Part I.

## 1. Why ASE Needs Its Own Solver

ASE is broadband, incoherent light generated by Yb¬≥‚Å∫ ions decaying spontaneously
and getting amplified as they propagate. Unlike the signal ‚Äî a coherent pulse at
exactly 1064 nm ‚Äî ASE:

1. **Spans a broad wavelength range** (roughly 970‚Äì1130 nm for Yb).
2. **Travels in BOTH directions** (forward, with the signal, and backward,
   toward the input).
3. **Eats inversion** ‚Äî every photon of ASE is built from an excited Yb ion, so
   ASE depletes the energy that should go into the signal pulse.
4. **Can lase parasitically** ‚Äî if facet reflections create round-trip gain > 1.

A scalar model that approximates ASE as a single power number with a fixed
bandwidth at 1064 nm misses every one of these. In particular it underestimates
true ASE in high-gain stages because (a) the actual gain peak is at ~1030 nm, not
1064 nm ‚Äî so the scalar model puts ASE at the wrong color; and (b) without
backward ASE, the inversion looks larger than it is, making gain look better than
it is.

The solver tracks ASE properly: 160 wavelength bins, both directions, all coupled
to the inversion equation.

---

## 2. The Spectral Grid ‚Äî Slicing the ASE Spectrum

We discretize 970‚Äì1130 nm into 160 bins, each 1 nm wide (Part I ¬ß3.1). Bin `i`
is centered at `Œª_i = Œª_min + (i + 0.5)¬∑ŒîŒª`, so bin 0 is at 970.5 nm and bin 159
at 1129.5 nm. Each bin has a center frequency `ŒΩ_i = c/Œª_i` and a frequency width
`ŒîŒΩ_i = c¬∑ŒîŒª/Œª_i¬≤`.

**Important:** `ŒîŒΩ_i` is NOT the same for every bin, because `ŒΩ = c/Œª` is
nonlinear. A 1 nm bin at 970 nm covers ~318 GHz; the same 1 nm bin at 1130 nm
covers ~235 GHz. The spontaneous-emission source (¬ß7) uses `ŒîŒΩ_i`, so this
matters for energy book-keeping.

The signal at 1064 nm and the pump at 976 nm are **separate channels**, not bins.

```
Total optical channels = 1 pump + 1 signal + 160 forward ASE + 160 backward ASE = 322
```

Every one of these 322 channels feeds into the inversion equation (¬ß5). Leaving
any of them out (the easy mistake) makes ASE depletion invisible to the inversion.

---

## 3. Cross-Sections ‚Äî How Strongly Yb Absorbs and Emits at Each Wavelength

Two wavelength-dependent quantities govern how Yb interacts with light:

- **œÉ_a(Œª)** = absorption cross-section ‚Äî how strongly a ground-state ion absorbs
  a photon at Œª.
- **œÉ_e(Œª)** = emission cross-section ‚Äî how strongly an excited ion is stimulated
  to emit at Œª.

Reference values are in Part I ¬ß3.2. Three things to notice:

1. **Both cross-sections are huge at 976 nm (the zero line)** ‚Äî the pump
   wavelength, strongly absorbed.
2. **œÉ_e dominates œÉ_a, and œÉ_e rises 1000 ‚Üí 1020 nm** ‚Äî the emission shoulder
   peaks just past 1020 nm, which is why that region has the largest gain ratio
   œÉ_e/œÉ_a.
3. **Below 1000 nm, œÉ_a is enormous** ‚Äî ASE generated near 976 nm gets
   reabsorbed almost immediately, producing the "dip" in the backward ASE
   spectrum.

The values live in `ase/data/cross_sections_yb.csv` (measured Melkumov AS table)
and are interpolated (piecewise log-linear, monotone between measured nodes) onto
each amplifier's spectral grid at startup, giving each fiber per-bin œÉ_a, œÉ_e
arrays of length 160.

---

## 4. Mode Overlap Œì(Œª) ‚Äî Why Per-Bin Values Matter

Even though all 160 bins propagate in the same fiber, they don't see the same
mode. A photon at 970 nm has a tighter mode than one at 1130 nm, because the
V-number depends on wavelength:

```
V(Œª) = (2œÄ / Œª) ¬∑ r_core ¬∑ NA
```

Smaller Œª ‚Üí larger V ‚Üí tighter mode ‚Üí larger overlap with the doped core. For
each bin we compute the mode radius `w(Œª)` from the Marcuse approximation, then:

```
Œì(Œª)   = 1 - exp(-2 ¬∑ (r_core / w(Œª))¬≤)      (overlap with the doped core)
A_eff(Œª) = œÄ ¬∑ w(Œª)¬≤
```

Across the band Œì varies by ~10% and A_eff by ~16% ‚Äî enough to matter for
per-bin gain. The pump is different: it propagates in the cladding, so
`Œì_pump = A_core / A_clad` (very small ‚Äî that's why cladding pumping needs lots
of pump power).

---

## 5. The Inversion Equation ‚Äî Summed Over All 322 Channels

At every position z, the excited fraction `n‚ÇÇ(z)` is set by the balance of
pumping-up and decaying-down (Part I ¬ß2.5):

```
n‚ÇÇ(z) = numerator(z) / [denominator(z) + 1/œÑ]
```

where each optical channel `k` contributes:

```
numerator term:    œÉ_a,k ¬∑ Œì_k ¬∑ P_k(z) / (h ¬∑ ŒΩ_k ¬∑ A_core)        ‚Üê drives n‚ÇÇ UP (absorption)
denominator term:  (œÉ_a,k + œÉ_e,k) ¬∑ Œì_k ¬∑ P_k(z) / (h ¬∑ ŒΩ_k ¬∑ A_core)  ‚Üê saturation
```

Summed over all 322 channels. Constants: `h` = 6.626 √ó 10‚Åª¬≥‚Å¥ J¬∑s,
`œÑ` = 0.83 ms (Yb upper-state lifetime), `A_core` (the doped area ‚Äî see ¬ß6).

**The crucial point:** every channel ‚Äî including all 160 backward ASE bins ‚Äî
appears in both sums. This is what makes ASE actually deplete the inversion. Omit
the ASE bins (as the old scalar model effectively did) and n‚ÇÇ(z) comes out too
high, so the simulated gain is unrealistically large.

---

## 6. A Convention Note: A_clad vs A_core for the Pump

A na√Øve reading would set "A_k = A_clad for the pump channel." Combined with
Œì_pump = A_core/A_clad, that double-counts the geometric dilution and gives a
pump rate ~1000√ó too small.

The textbook formulation (Becker et al.; Snitzer & Po) uses **A_core for ALL
channels**, with Œì_pump = A_core/A_clad expressing the cladding-mode overlap. For
the pump this simplifies cleanly:

```
œÉ_a ¬∑ Œì_pump ¬∑ P / (h ¬∑ ŒΩ ¬∑ A_core)
  = œÉ_a ¬∑ (A_core/A_clad) ¬∑ P / (h ¬∑ ŒΩ ¬∑ A_core)
  = œÉ_a ¬∑ P / (h ¬∑ ŒΩ ¬∑ A_clad)              ‚Üê the standard cladding-pumped result
```

Same answer, no double-counting. The simulator uses `A_dope = A_core`
everywhere; this matches the scalar `components.py` rate equation and produces
physically correct inversions. The choice is documented in
`ase/solver_steady.py:_compute_n2_local`.

---

## 7. Spontaneous Emission Source ‚Äî Where ASE Comes From

Each ASE bin has a built-in source: photons spontaneously emitted by excited ions
that couple into that bin's guided mode (Part I ¬ß2.4):

```
S_k(z) = m_pol ¬∑ h ¬∑ ŒΩ_k ¬∑ ŒîŒΩ_k ¬∑ Œì_k ¬∑ œÉ_e,k ¬∑ N_Yb ¬∑ n‚ÇÇ(z)
```

- `m_pol = 2` for non-PM fiber (two polarizations) or `1` for PM.
- `h ¬∑ ŒΩ_k` is the energy per photon.
- `ŒîŒΩ_k` is the bandwidth this bin captures ‚Äî wider bin ‚Üí more photons.
- `Œì_k ¬∑ œÉ_e,k ¬∑ N_Yb ¬∑ n‚ÇÇ(z)` is the per-volume spontaneous emission rate into
  the mode.

Pump and signal channels do NOT have this source ‚Äî they're externally injected.
Forward and backward ASE share the same source magnitude (spontaneous emission is
isotropic).

---

## 8. The Propagation Equations ‚Äî How Powers Change Along z

For each channel k with direction `u_k` (+1 forward, -1 backward), in steady
state:

```
u_k ¬∑ dP_k/dz = u_k ¬∑ g_k(z) ¬∑ P_k(z)   (gain/absorption)
              + u_k ¬∑ S_k(z)            (spontaneous emission, ASE bins only)
              - u_k ¬∑ Œ±_bg ¬∑ P_k(z)     (background loss, ~0.005 dB/m)
```

with the gain coefficient

```
g_k(z) = Œì_k ¬∑ N_Yb ¬∑ [n‚ÇÇ(z) ¬∑ œÉ_e,k - (1 - n‚ÇÇ(z)) ¬∑ œÉ_a,k]
```

A channel is amplified when n‚ÇÇ is above its transparency point
`n‚ÇÇ_transparent,k = œÉ_a,k / (œÉ_a,k + œÉ_e,k)`. For the signal at 1064 nm this is
~3%, so almost any inversion gives signal gain. For a 975 nm bin it's ~50%, so
you need n‚ÇÇ above 50% before that bin amplifies rather than absorbs ‚Äî which is
why backward ASE shows the 975 nm dip even when the fiber is well inverted.

```
Forward (u_k = +1):   dP_fwd/dz = +(g_k - Œ±_bg) ¬∑ P_fwd + S_k     (S only for ASE bins)
Backward (u_k = -1):  dP_bwd/dz = -(g_k - Œ±_bg) ¬∑ P_bwd - S_k
```

---

## 9. Boundary Conditions ‚Äî Why This Is a BVP

Some powers are known at the input (z = 0), others at the output (z = L); none
fully known at both. That makes it a **boundary-value problem**.

```
At z = 0:  P_pump(0), P_signal(0), P_ASE_fwd(0, Œª) known;  P_ASE_bwd(0, Œª) UNKNOWN
At z = L:  P_ASE_bwd(L, Œª) = R_out ¬∑ P_ASE_fwd(L, Œª)        (Fresnel reflection)
```

`R_out` is the output power reflectivity (‚âà 3.5% flat-cleaved; < 10‚Åª‚Å¥
angle-cleaved ‚Äî the default; < 10‚Åª¬≥ AR-coated). Forward and backward fields are
coupled (backward ASE depletes the inversion the forward fields see, and forward
ASE feeds the backward field at z = L through the reflection), so solving them
requires iteration.

---

## 10. The Iterative-Shooting Solver ‚Äî One Iteration at a Time

The solver lives in `ase/solver_steady.py` (Part I ¬ß4.1):

```
Initialize fields (P_ase_bwd seeded from spontaneous emission, NOT zero ‚Äî ¬ß11).

Repeat until converged:
    1. Forward sweep z=0 ‚Üí z=L: RK4-step the pump, signal, and forward-ASE
       channels, recomputing n‚ÇÇ(z) LOCALLY from the just-updated P at each step.
    2. Apply boundary condition at z=L:  P_ase_bwd(L, Œª) = R_out ¬∑ P_ase_fwd(L, Œª).
    3. Backward sweep z=L ‚Üí z=0: RK4-step the backward-ASE channels, again with
       the local-n‚ÇÇ update.
    4. Convergence: max relative change in any channel < 1e-5.

Return n‚ÇÇ(z), P_pump(z), P_signal(z), P_ase_fwd(z, Œª), P_ase_bwd(z, Œª).
```

The key trick is the local-n‚ÇÇ update inside each sweep (step 1), which deserves
its own section.

---

## 11. Why Local n‚ÇÇ Prevents Runaway

A na√Øve implementation freezes n‚ÇÇ(z) for the whole iteration: compute it once,
then sweep forward and backward without changing it. That fails badly in
high-gain stages.

Consider a well-inverted high-gain stage. From a "constant pump, no ASE" trial,
the inversion comes out well above transparency everywhere, giving a large gain
coefficient at the 1030 nm peak ‚Äî tens of dB per meter. Over a few meters that's
a hundred-odd dB of single-pass gain, so a tiny spontaneous source becomes
astronomically amplified after one sweep, and the backward ASE explodes too.
Numerically the run blows up; physically the system would have saturated long
before.

The fix is to update n‚ÇÇ at every step of the sweep using the *just-computed*
local P values. As ASE grows along z, n‚ÇÇ at the new z drops to reflect the new
saturation, and the next step uses that smaller gain coefficient. The exponential
blow-up self-limits ‚Äî saturation is built into the sweep, not deferred to the
next iteration. This is what lets the solver converge in a handful of iterations;
without it, even heavy underrelaxation fails on a high-gain stage.

The initial guess itself is not zero-ASE, either: the default seeds the forward
**and backward** ASE fields from spontaneous emission on a pump-only inversion
profile (`_spontaneous_emission_init`). A zero-ASE start lets a high-gain fiber's
first sweep amplify a negligible source into a runaway and settle in the
unphysical ASE-dominated basin (¬ß13b); the spontaneous-emission floor lands the
iteration in the physical, signal-dominated basin instead.

## 12. Convergence and the Runaway Guard

At the end of each iteration the solver computes the largest relative change in
any channel at any z:

```
max_rel = max over k, z of  |P_k_new(z) - P_k_old(z)| / max(|P_k_new(z)|, 1e-15)
```

If `max_rel < 1e-5` (after at least 2 iterations), it declares convergence. If it
hasn't converged after 100 iterations, it returns with `converged=False` and a
note.

There's also a **runaway sentinel**: if any single bin's power exceeds 1 kW
during an iteration, the solver bails, clamps the spectrum, and sets
`parasitic_lasing=True`. This catches configurations where no steady state
exists (¬ß13).

## 13. Parasitic Lasing ‚Äî When Steady State Doesn't Exist

If end-facet reflections create round-trip gain > 1 at *any* wavelength, the
system lases parasitically. The BVP has no steady solution; a real fiber would
build a lasing mode and oscillate. The condition (Part I ¬ß5.3):

```
G(Œª)¬≤ ¬∑ R_in ¬∑ R_out ‚â• 1     for any Œª,   where G(Œª) = exp(‚à´‚ÇÄ·¥∏ g(Œª, z) dz)
```

The solver checks this after each successful run using `G(Œª)` from the converged
n‚ÇÇ(z). If met, `parasitic_lasing=True` is set with a note giving the maximum
round-trip gain.

This is why angle-cleaved facets (R ‚âà 10‚Åª‚Å¥) matter: lowering R by orders of
magnitude is what keeps round-trip gain below 1 in a high-gain stage. Lengthening
the fiber or raising the pump both raise single-pass gain and can push a stage
over the line; the solver tells you when you've crossed it.

## 13b. The Layered Robust Solver ‚Äî Catching Non-Physical Fixed Points

`_check_parasitic_lasing` only fires when **both** `R_in ¬∑ R_out > 0`. There's a
subtler failure mode it can't catch on its own: when the gain is high enough that
the BVP admits multiple self-consistent fixed points, the iterative shooter can
silently land on one that violates global energy conservation.

### The failure mode (literature: well-known, not formally named)

The Giles-Desurvire propagation system at our resolution (322 channels, RK4) is a
multi-dimensional root-finding problem with strong exponential sensitivity.
Paschotta (RP Photonics), Ren et al. (Opt. Quantum Electron. 47(7), 2199, 2015),
and the `scipy.integrate.solve_bvp` documentation all describe the phenomenon:
high-gain BVPs with bidirectional ASE have **many basins of attraction**, and
Newton-type root finders converge to whichever one the initial guess lies in.
Local rate equations can be satisfied at every z to machine precision while the
*global* energy budget fails.

The global constraint is the **quantum-defect (QD) ceiling**: pump photons carry
more energy than signal photons, so

```
signal_gain_W ‚â§ (ŒΩ_signal / ŒΩ_pump) ¬∑ pump_absorbed_W ‚â° QD ¬∑ pump_absorbed_W,   QD ‚â§ 1.
```

The difference `h¬∑ŒΩ_pump ‚àí h¬∑ŒΩ_signal` per transition is quantum-defect heat
deposited in the lattice. The rate equation enforces per-ion photon balance
locally; it does **not** automatically enforce the global energy budget. In
healthy regimes the BVP has a unique fixed point and the iteration finds it; near
the high-gain edge it can converge to an ASE-dominated basin where the reported
signal output exceeds the QD ceiling by several times ‚Äî mathematically
self-consistent, physically impossible. (The spontaneous-emission seed of ¬ß11 is
the first line of defense; the layers below are the safety net for what it
doesn't rescue.)

### The remedy ‚Äî three layers + three diagnostics

The robust entry point `ase/solver_time.py::solve_steady_state_robust` wraps the
raw Mode A solver in three cascading layers (also summarized in Part I ¬ß4.4):

**Layer 1: direct solve + health classification.** Compute
`signal_gain + ASE_out + bg_loss` and compare to `QD ¬∑ pump_absorbed`. Within 1%
of (or under) the ceiling ‚Üí return. 1‚Äì5% over ‚Üí return with a warning. >5% ‚Üí
Layer 2. The 1%/5% thresholds are **engineering judgement** (no published
canonical value): ~20√ó the RK4-truncation + iteration noise floor (~10‚Åª¬≥) and
well above the worst observed healthy regime (a fully saturated power stage runs
just under the ceiling).

**Layer 2: Ren et al. 2015 homotopy continuation.** Walk from a known-easy
operating point to the target, each step warm-starting the next via the `init=`
parameter:

1. Pump-only solve (`signal=0`, `m_pol=0` to disable the ASE source). Unique
   fixed point.
2. Add signal, keep ASE source off. Beer-Lambert-like ‚Äî still unique.
3. Enable ASE source. **Critical step**: the iteration starts in the
   signal-clamped basin instead of the ASE-dominated one.
4. Parameter continuation: scale pump 50% ‚Üí 75% ‚Üí 100% of target, each warm-started.
5. Final fallback: Xu et al. 2014 linear-gain-shape multistart with three slope
   efficiencies (0.3, 0.5, 0.7).

References: Opt. Quantum Electron. 47(7), 2199 (2015),
DOI 10.1007/s11082-014-0096-8; Optik 2014 pii S1068520014000546.

**Layer 3: time-marching B1+B2 arbiter.** If Layer 2 still violates, the system
likely has no physical CW steady state. Dispatch to
`solve_time_dependent(mode="full")` (B1 rate-equation time-stepping + B2
Lax-Wendroff pulse PDE). The time-dependent IVP has a **unique attractor** ‚Äî
landing on an energy-violating fixed point is impossible by construction. This is
the pattern PyFiberAmp uses with `DynamicSimulation(stop_at_steady_state=True)`.
If B1+B2 itself bails (parasitic guard fires), no physical steady state exists at
all; the wrapper returns Mode A's runaway-clamped result with
`solver_path_used="all_failed"`.

> **Circular-dependency note.** Layer 3 calls `solve_time_dependent(mode="full")`,
> which calls `_periodic_steady_state`, whose warmup must NOT call the full
> robust wrapper (that would recurse into Layer 3). The warmup therefore calls
> `solve_steady_state_homotopy` directly ‚Äî homotopy alone never invokes the
> time-dependent solver. A separate NaN-guard inside `_periodic_steady_state`
> remains as a defensive backstop for genuinely past-parasitic configurations.

### The three diagnostics

In `ase/solver_health.py`:

| Diagnostic | Formula | Threshold(s) | Reference |
|---|---|---|---|
| `energy_residual_ratio` | `(consumed - delivered) / delivered`, with `delivered = QD ¬∑ pump_absorbed`, `consumed = signal_gain + ASE_out + bg_loss` | `> 0.01` warn, `> 0.05` violate | Ren et al. 2015 ¬ß2 (photon balance); engineering judgement on thresholds |
| `ase_conversion_fraction` (Œ∑_ASE) | `ASE_out_total / pump_absorbed` | `> 0.1` "mixed", `> 0.3` "sfs" | Wang & Clarkson 2006 (110 W Yb SFS); Dong 2025 operational thresholds |
| `small_signal_g0L` | `Œì_signal ¬∑ N ¬∑ (n2_asymp¬∑œÉ_e,sig ‚àí (1‚àín2_asymp)¬∑œÉ_a,sig) ¬∑ L` | `> 30` flag "high_gain" | Furuse et al. used `g(0)¬∑l ‚âà 3` for Yb:YAG thin-disk; the fibre threshold is ~10√ó higher because the doped core covers only a small fraction of the mode |

### What the user sees

`Amplifier.info` gains seven keys after every steady-state solve:

```
"energy_residual_ratio":   float,
"ase_conversion_fraction": float,    # Œ∑_ASE
"small_signal_g0L":        float,
"energy_status":           "ok" | "warning" | "violation",
"regime":                  "amplifier" | "mixed" | "sfs" | "high_gain",
"solver_path_used":        "direct" | "homotopy" | "time_marching_arbiter" | "all_failed",
"homotopy_steps_used":     int,
```

The text report adds a compact `[E:.. R:.. path=..]` decoration on the gain line
whenever the solve is **not** in the clean "direct + healthy + amplifier" case
(e.g. `[E:WARNING √ó1.01  R:AMPLIFIER  path=homotopy(4)]` or
`[E:VIOLATION √ó5.30  R:SFS  path=all_failed]`).

### When you see `[E:VIOLATION]`

The simulator is telling you: (1) the BVP solved to tolerance ‚Äî the numbers are a
*self-consistent* local solution; (2) none of direct ‚Üí homotopy ‚Üí time-marching
found a physical solution respecting the QD ceiling; (3) in a real lab this
configuration would not act like a clean amplifier ‚Äî it would parasitically lase
or become a superfluorescent source; (4) the action is to change the design
(shorten the fibre, reduce pump, increase seed) until the decoration goes away.

### Cost

In a healthy regime, Layer 1 passes on the first attempt ‚Äî the wrapper is a thin
pass-through with one extra diagnostics call (~3% overhead). In marginal regimes,
Layer 2's continuation adds a few hundred ms. If Layer 3 fires, time-marching
adds seconds, but only for configurations on the edge of physicality.

## 14. Bandpass Filters Between Stages

The default system places a Gaussian bandpass filter (modeling the FBG) just
after each post-amplifier isolator:

```
Center wavelength:  1064 nm
FWHM:               2 nm
Peak transmission:  ~ -0.5 dB insertion loss
```

The transfer function for ASE is:

```
T(Œª) = 10^(-peak_loss_dB / 10) ¬∑ exp(-4 ¬∑ ln(2) ¬∑ ((Œª - 1064 nm) / FWHM)¬≤)
```

At 1064 nm the filter passes the peak (just the insertion loss); 34 nm off-band
at 1030 nm the ideal Gaussian is effectively zero (real FBGs floor at ~50 dB
rejection). Either way the 1030 nm gain peak is crushed before the next stage, so
downstream amplifiers only re-amplify the small fraction of ASE near the signal
wavelength ‚Äî where the gain is already clamped by signal saturation, keeping the
noise figure low. The filter applies `T(Œª)` to the ASE spectrum and a flat
insertion loss to the coherent signal (which sits exactly at 1064 nm, so it sees
only the peak loss).

## 15. Mode A vs Mode B ‚Äî When to Use Each

The amplifier supports two solver modes via the `mode=` argument to
`Amplifier.propagate()`:

**Mode A ‚Äî `"steady"`.** Quasi-CW: solve the steady-state BVP using the
time-averaged signal power. Exact when the rep-rate period is much shorter than
the upper-state lifetime.

**Mode B ‚Äî `"time-dependent"` (the default).** Level 5 ‚Äî the two-part MOPA
decomposition:
- **B1** (inter-pulse recovery) ‚Äî adaptive rate-equation time-stepping; pumps up
  the inversion while ASE grows. **All ASE accounting happens here.**
- **B2** (pulse propagation) ‚Äî Lax-Wendroff (z, t) at CFL = 1, propagating the
  **signal pulse and pump only** (ASE neglected on the ns scale). Captures
  pulse-shape distortion and gain saturation as the pulse extracts the inversion
  B1 prepared.

Auto-dispatch picks the branch by rep rate:
- *High rep* (period < 0.1¬∑œÑ): delegates to Mode A ‚Äî exact when n‚ÇÇ barely changes
  between pulses. Same numbers, same wall time.
- *Otherwise*: runs the B1 + B2 cycle, warm-started from a Mode A solve, iterating
  until pre-pulse n‚ÇÇ(z) is stable (`cycles_tol = 1e-3`, `cycles_max = 25`).

| Rep rate | Period | Period / œÑ | Mode B branch |
| -------- | ------ | ---------- | ------------- |
| 100 kHz | 10 ¬µs | 0.012 | high_rep_quasi_cw (= Mode A) |
| 10 kHz | 100 ¬µs | 0.12 | periodic (B1+B2) |
| 1 kHz | 1 ms | 1.2 | periodic |
| 100 Hz | 10 ms | 12 | periodic |
| 10 Hz | 100 ms | 119 | periodic |

To **force** the B1+B2 path at any rep rate (for pulse-shape distortion studies),
pass `--force-b2` to `simulate.py` or `mode="full"` to `Amplifier.propagate()`.
With `--plots`, this also renders `pulse_shape.png` (temporal profile at z=0 vs
z=L per amplifier). The Level 5 benchmarks are covered in
`ase/tests/test_time_solver.py`.

## 16. Frantz-Nodvik Extraction ‚Äî How Mode B Handles the Pulse

In Mode B's low-rep / periodic branches, the pulse extracts energy slice-by-slice
using the Frantz-Nodvik formula. For each slice [z[i], z[i+1]]:

```
E_sat   = h ¬∑ ŒΩ_signal ¬∑ A_eff_signal / (œÉ_e_signal ¬∑ Œì_signal)
g0(i)   = Œì_signal ¬∑ N_Yb ¬∑ (n‚ÇÇ(z[i])¬∑œÉ_e - (1-n‚ÇÇ(z[i]))¬∑œÉ_a)
G0(i)   = exp(g0(i) ¬∑ dz)
E_out   = E_sat ¬∑ ln(1 + G0(i) ¬∑ (exp(E_in / E_sat) - 1))
```

The extracted energy is `E_out - E_in`; the slice's inversion drops by
`Œîn‚ÇÇ(i) = (E_out - E_in) / (h ¬∑ ŒΩ_signal ¬∑ A_core ¬∑ dz ¬∑ N_Yb)`. Then
`E_in_next = E_out` and the process repeats to z = L. At low rep rate the
inversion has time to fully recover between pulses, so each pulse can extract a
large fraction of the stored energy ‚Äî far more than the seed pulse energy. The
B2 Lax-Wendroff path additionally captures pulse-shape distortion that the
per-slice extraction does not.

## 17. What Changed vs. the Old Scalar Model ‚Äî At a Glance

| Aspect | Old (scalar) | New (BVP) |
| ------ | ------------ | --------- |
| ASE representation | Single number `P_ASE` | 160 wavelength bins (forward) + 160 (backward) |
| ASE wavelength assumption | Always at 1064 nm | Computed across 970‚Äì1130 nm |
| ASE bandwidth | Hardcoded ~5 THz | Per-bin ŒîŒΩ, bin-width-aware |
| Backward ASE | Not modeled | Solved as part of the BVP |
| Inversion equation | Sums pump and signal only | Sums all 322 channels |
| Parasitic lasing | Not detected | Detected and flagged |
| Inter-stage filtering | Flat dB loss only | Wavelength-dependent transfer function |
| Cross-section data | A few hardcoded scalars | Measured CSV interpolated to fiber grid |
| Solver | Closed-form, single forward sweep | Iterative shooting with local-n‚ÇÇ update |

The architectural addition is the `ase/` package; `components.py` and
`simulate.py` keep the same shape but now exchange `OpticalState` (signal + ASE
state) instead of bare `Signal` objects. The new model is more pessimistic about
ASE in high-gain stages, which is a *correctness* improvement ‚Äî that ASE was
always there; the scalar model just hid it.
