# ASE Documentation — Specification & Walkthrough

This is the single source of truth for Amplified Spontaneous Emission (ASE)
modeling in the simulator: the physics, equations, algorithms, data structures,
and validation, plus a plain-language walkthrough of how the solver actually
works.

It is organized in two parts:

- **Part I — Specification.** The rigorous reference: every equation, the
  spectral discretization, the bidirectional BVP, time-dependent pulse dynamics,
  inter-stage coupling, data structures, validation benchmarks, and common
  pitfalls. **Read this fully before writing or modifying any ASE code.**
- **Part II — Walkthrough.** A step-by-step, no-programming-required explanation
  of how the spectrally-resolved solver computes ASE inside each amplifier
  stage: the 322 channels, how the inversion sums over all of them, why the
  local-n₂ update prevents runaway, Mode A vs Mode B, and the layered robust
  solver. Read it alongside Part I.

The ASE solver lives in the `ase/` package; this document is its companion.
For gain, nonlinear effects, and the system-level chain, see
`computation_walkthrough.md` and `physics_spec.md`.

> **Numbers policy.** This document contains physics constants, cross-section
> data, equations, and grid/structure definitions — all system-independent.
> It deliberately does **not** quote simulation *outputs* (gains, output
> powers, ASE levels, iteration counts) because those depend on the specific
> system being simulated. To see live values for a given configuration, run
> `python simulate.py`.

# Part I — Specification

## 1. What ASE Is and Why It Matters

Amplified Spontaneous Emission is light generated by spontaneous decay of
excited Yb³⁺ ions that then gets amplified as it propagates through the inverted
fiber. Unlike the signal (a coherent pulse at 1064 nm), ASE is broadband
(~970–1130 nm), incoherent, propagates in both directions, and acts as
broadband noise that:

1. **Depletes the inversion** — ASE consumes stored energy that should go to the
   signal pulse.
2. **Limits achievable gain** — at high enough gain, ASE clamps the inversion
   (ASE-limited gain).
3. **Degrades pulse energy at low rep rates** — between pulses, ASE builds up and
   eats stored energy.
5. **Adds noise to the output** — forward ASE co-propagates with the signal.

A simulator that estimates ASE as a single number (like
`P_ASE ≈ n_sp · (G-1) · hν · Δν`) misses all of the above dynamics. Accurate ASE
modeling requires spectrally resolved, bidirectional propagation coupled to the
time-dependent rate equations.

## 2. The Equation Set

### 2.1 Variables

At every position z along the fiber and time t:

```
n₂(z, t)    — fraction of Yb ions in the excited state (0 ≤ n₂ ≤ 1)
N            — total Yb³⁺ ion density [ions/m³] (constant, derived from cladding absorption)
N₂ = n₂ · N  — excited-state population density
N₁ = (1 - n₂) · N  — ground-state population density

P_pump(z, t)       — pump power [W] (forward or backward)
P_signal(z, t)     — signal power [W] (forward)
P_ASE_fwd(z, t, λ) — forward ASE power in wavelength bin centered at λ [W]
P_ASE_bwd(z, t, λ) — backward ASE power in wavelength bin centered at λ [W]
```

### 2.2 Propagation equations

For each optical channel k with direction u_k = +1 (forward) or -1 (backward):

```
(1/v_g) ∂P_k/∂t + u_k · ∂P_k/∂z = u_k · g_k(z,t) · P_k(z,t)
                                    + u_k · S_k(z,t)
                                    - u_k · α_bg · P_k(z,t)
```

Where:
- `v_g = c / n_glass` — group velocity in fiber (~2.07 × 10⁸ m/s for n_glass = 1.45)
- `g_k(z,t)` — gain/absorption coefficient for channel k
- `S_k(z,t)` — spontaneous emission source (only nonzero for ASE channels)
- `α_bg` — background fiber loss (~0.005 dB/m ≈ 0.00115 Np/m at 1064 nm)

### 2.3 Gain coefficient

```
g_k(z,t) = Γ_k · N · [n₂(z,t) · σ_e(λ_k) - (1 - n₂(z,t)) · σ_a(λ_k)]
```

Where:
- `Γ_k` — overlap factor between mode at wavelength λ_k and doped core
- `σ_e(λ_k)` — emission cross-section at wavelength λ_k [m²]
- `σ_a(λ_k)` — absorption cross-section at wavelength λ_k [m²]

For the **pump** channel in a double-clad fiber:
```
Γ_pump = A_core / A_clad
```

For the **signal** and **ASE** channels (core-guided):
```
Γ_k = 1 - exp(-2 · (r_core / w(λ_k))²)
```
where `w(λ_k)` is the mode radius at that wavelength, computed from the Marcuse
approximation with wavelength-dependent V-number:
```
V(λ) = (2π / λ) · r_core · NA
w(λ) / r_core = 0.65 + 1.619 / V(λ)^1.5 + 2.879 / V(λ)^6    (if V < 2.405)
w(λ) = 0.65 · r_core                                           (if V > 2.405, fundamental mode)
```

**Implementation note**: Pre-compute `Γ_k`, `σ_e(λ_k)`, `σ_a(λ_k)` for every ASE
bin once at initialization. Store as arrays indexed by bin number. These do not
change during simulation.

### 2.4 Spontaneous emission source term

```
S_k(z,t) = m_pol · h · ν_k · Δν_k · Γ_k · σ_e(λ_k) · N · n₂(z,t)
```

Where:
- `m_pol = 1` for PM (polarization-maintaining) fiber
- `m_pol = 2` for non-PM fiber (two orthogonal polarization modes)
- `h = 6.626 × 10⁻³⁴` J·s
- `ν_k = c / λ_k` — frequency of ASE bin k [Hz]
- `Δν_k` — width of ASE bin k in Hz: `Δν_k = c · Δλ / λ_k²`

This term is the seed that starts ASE. It represents the fraction of
spontaneously emitted photons that couple into the guided mode. One source term
per ASE bin, per direction (forward and backward get identical source terms —
spontaneous emission is isotropic in the guided mode).

**The source term is NOT applied to the pump or signal channels** — only to ASE bins.

### 2.5 Rate equation for upper-state population

```
∂n₂/∂t = R_abs(z,t) - R_em(z,t) - n₂/τ
```

Where the absorption and emission rates sum over ALL channels (pump + signal +
every ASE bin in both directions):

```
R_abs = Σ_k [ σ_a(λ_k) · Γ_k · P_k(z,t) / (h · ν_k · A_k) ] · (1 - n₂)

R_em  = Σ_k [ σ_e(λ_k) · Γ_k · P_k(z,t) / (h · ν_k · A_k) ] · n₂
```

Where `A_k` is the area the dopant ions occupy. **The implementation uses
`A_k = A_core` for ALL channels, including the pump** (with Γ_pump = A_core/A_clad
expressing the cladding-mode overlap). This is the textbook convention (Becker
et al.; Snitzer & Po) and avoids double-counting the pump's geometric dilution —
see Part II §6 for the derivation. The lifetime term `- n₂/τ` is spontaneous
decay (τ = 0.83 ms for Yb, the measured Melkumov AS value).

**Steady-state form** (set ∂n₂/∂t = 0, solve for n₂):

```
n₂(z) = [Σ_k σ_a(λ_k) · Γ_k · P_k / (h·ν_k·A_core)]
       / [Σ_k (σ_a(λ_k) + σ_e(λ_k)) · Γ_k · P_k / (h·ν_k·A_core) + 1/τ]
```

This is used in the inter-pulse BVP solver and for CW analysis.

## 3. Spectral Discretization

### 3.1 Wavelength grid

```
λ_min  = 970 nm     (must include the 975 nm absorption/emission peak)
λ_max  = 1130 nm    (tail of Yb emission)
Δλ     = 1 nm       (standard resolution)
N_bins = (λ_max - λ_min) / Δλ = 160 bins
```

Each bin has a center wavelength `λ_i = λ_min + (i + 0.5) · Δλ` and frequency
width `Δν_i = c · Δλ / λ_i²`.

**The signal channel at 1064 nm is a separate channel, not an ASE bin.** It has
its own power variable with different initial conditions and no spontaneous
emission source. The pump at 976 nm is a separate channel too.

Total optical channels: 1 pump + 1 signal + 160 forward ASE + 160 backward
ASE = **322 channels**.

> **Grid note (915 nm pumping).** The pump/signal scalar channels resolve any
> wavelength inside the cross-section data range (848–1180 nm), so 915 nm
> pumping works. The ASE *bin* grid default stays at [970, 1130] nm — widening
> `lambda_min` changes `n_bins` and every ASE result, so it is left as an
> explicit caller choice in `spectral_grid.from_fiber`.

### 3.2 Cross-section spectrum data

`σ_a(λ)` and `σ_e(λ)` are needed as arrays sampled at each bin center. The
simulator ships the **measured aluminosilicate (AS) cross-section table** of
Melkumov et al. in `ase/data/cross_sections_yb.csv` — the closest published
host-glass match for the Coherent/Nufern SM-YDF-5/130-VIII fiber. The table is
the measured source of truth; it is *not* generated from anchors.

> **Provenance.** The full numeric table comes from the **non-peer-reviewed
> preprint** (M.A. Melkumov, I.A. Bufetov, K.S. Kravtsov, A.V. Shubin,
> E.M. Dianov, *"Absorption and emission cross section of Yb3+ ions in Al2O3
> and P2O5 doped fibers,"* FORC Preprint No. 5, Moscow, 2004; arXiv:1502.02885,
> Appendix 2, AS columns). A **peer-reviewed** English/Russian journal version
> of the same study exists (Mel'kumov et al., *"Lasing parameters of
> ytterbium-doped fibres doped with P2O5 and Al2O3,"* Quantum Electronics
> 34(9), 843–848 (2004), DOI 10.1070/QE2004v034n09ABEH002688) but is a shorter
> presentation that does **not** reproduce the Appendix-2 table — so the table
> itself is not peer-reviewed, though a peer-reviewed version of the work
> exists. Published units are pm² (= 10⁻²⁴ m²); the CSV stores m² (× 1e-24).
> Lifetime τ = 0.83 ms (AS). The raw published-pm² transcription is kept at
> `ase/data/melkumov_AS_raw.csv` for provenance.

Representative measured AS values (from the CSV; the absolute scale cancels under
the N back-derivation in §4.4 of `physics_spec.md`, so only the spectral *shape*
matters physically):

```
λ [nm]    σ_a [10⁻²⁵ m²]    σ_e [10⁻²⁵ m²]
  915        6.5                 0.22
  940        3.2                 0.53
  976       26.9                29.7          ← zero line (σ_e/σ_a ≈ 1.10)
  1000       0.99                3.6
  1010       0.86                4.9
  1020       0.70                6.5
  1030       0.43                6.2          ← emission shoulder peak (>1020 nm)
  1064       0.046               3.0          ← signal wavelength (σ_e/σ_a ≈ 65)
  1080       0.018               2.3
  1100       0.0056              1.6
  1130       ~0.0001             ~0.06
```

Note σ_e(1030) > σ_e(1010) > σ_e(1000): the real Yb:silica emission shoulder
peaks just past 1020 nm.

**Interpolation: piecewise log-linear, monotone between measured nodes.** The
data is interpolated onto each fiber's spectral grid at startup; log-linear is
monotone between nodes by construction → zero overshoot near the narrow 976 nm
peak (FWHM ≈ 7.7 nm) and the 1030 nm shoulder. σ is clamped to a positive floor
before `log()`. A single shared helper (`cross_sections._log_linear_interp`) is
used by both `cross_sections.py` and `dopants.py` so the two paths cannot
diverge. Out-of-range wavelengths raise `ValueError` (no extrapolation).

**Critical**: the 970–1000 nm region has LARGE absorption cross-sections. ASE
generated near 975 nm (where σ_e is huge) gets strongly reabsorbed near the
pump-input end of the fiber. This spectral reshaping is a key physical effect
that only a spectrally resolved model captures.

### 3.3 Convergence check

Verify convergence by halving Δλ (to 0.5 nm, 320 bins) and confirming that total
integrated ASE power changes by a small amount (the test suite asserts < 5%).

## 4. Solver Architecture

The simulation operates in two modes that hand off to each other.

### 4.1 Mode A: Steady-state BVP solver (inter-pulse equilibrium)

**When to use**: computing the inversion profile and ASE distribution either
(a) at CW equilibrium, or (b) at the end of a long inter-pulse interval when the
system has reached quasi-steady-state.

**What it solves**: the propagation equations with ∂/∂t = 0:

```
u_k · dP_k/dz = u_k · g_k(z) · P_k(z) + u_k · S_k(z) - u_k · α_bg · P_k(z)
```

plus the steady-state inversion (§2.5).

**Boundary conditions**:

```
At z = 0 (input end):
    P_pump_fwd(0)    = P_pump_input            (known)
    P_signal_fwd(0)  = P_signal_input          (known)
    P_ASE_fwd(0, λ)  = P_ASE_input_fwd(λ)      (known — zero for first stage, or output of previous stage)
    P_ASE_bwd(0, λ)  = ???                     (UNKNOWN — this is what we solve for)

At z = L (output end):
    P_ASE_bwd(L, λ)  = R_output · P_ASE_fwd(L, λ)  (reflection; R < 10⁻⁴ for angle-cleaved — see §5.3)
    P_pump_bwd(L)    = R_output · P_pump_fwd(L)     (if counter-pumped, set separately)
```

This is a two-point BVP because forward channels are known at z=0, backward
channels are known at z=L.

**Algorithm: iterative shooting**

```
1. Initialize: seed P_ASE_bwd(0, λ) (and P_ASE_fwd) from spontaneous emission
   on the pump-only inversion profile — NOT zero. (See note below.)
2. Loop until converged:
   a. Compute n₂(z) at every grid point from current {P_k(z)}
   b. Integrate forward channels from z=0 to z=L using RK4
   c. Apply boundary condition at z=L: P_ASE_bwd(L) = R · P_ASE_fwd(L)
   d. Integrate backward channels from z=L to z=0 using RK4
   e. Check convergence: max |P_k_new - P_k_old| / P_k_old < tol for all k
   f. If not converged, update n₂(z) and repeat from (b)
3. Return: n₂(z), all P_k(z) profiles
```

Convergence tolerance: `tol = 1e-5` (relative). Spatial grid: 100–500 points per
fiber (default 200). See Part II §10–§12 for why the **local-n₂ update inside
each sweep** (not a frozen-n₂ iteration) is what keeps high-gain stages stable.

**Initial-guess note (spontaneous-emission seed).** Step 1 does *not* start the
backward ASE at zero. A constant-power, zero-ASE trial lets a high-gain fiber
amplify a negligible source into a runaway on the first sweep and converge to an
unphysical ASE-dominated fixed point. Instead, a single pump-only forward sweep
(signal = 0, ASE source off) yields an exact pump-only inversion n₂(z); the
spontaneous-emission source evaluated on that profile is accumulated gain-free
from each facet to seed both the forward and backward ASE fields. This
physically-motivated lower-bound floor lands the iteration in the physical
(signal-dominated) basin. It degrades gracefully — at zero pump the inversion is
zero, so the seed collapses to the historical zero-ASE guess. Implemented in
`ase/solver_steady.py::_spontaneous_emission_init`; a caller may still override
it via the `init=` parameter (used by the homotopy warm-starts, §4.4 / Part II
§13b).

**Alternative: scipy.integrate.solve_bvp.** The same problem can be posed as a
relaxation problem and handed to `solve_bvp` (collocation, Kierzenka & Shampine
2001). It is more robust than shooting for high-gain systems near the
runaway limit, at higher cost. The simulator uses iterative shooting with the
local-n₂ update; `solve_bvp` is noted as the evaluated-but-not-adopted
alternative.

### 4.2 Mode B: Time-dependent pulse propagation

**When to use**: simulating (a) the ns pulse passing through the fiber, and
(b) the inter-pulse inversion recovery with ASE buildup.

**What it solves**: the full time-dependent equations from §2.

#### Sub-regime B1: Inter-pulse recovery (µs–ms timescale)

Between pulses, there is no signal. The pump rebuilds n₂(z) while ASE grows. The
optical transit time (~5 ns/m) is negligible compared to the evolution
timescale, so the `(1/v_g) ∂P/∂t` term is dropped. **All ASE accounting happens
here.** Operator-split: the spatial P-field is re-solved (one forward + backward
sweep at fixed n₂) only when n₂ has drifted by >5% from the last solve; in
between, the n₂ rate equation is integrated forward in time with adaptive dt
(10 ns–10 µs, target ≤ 1% Δn₂ per step).

**Simplifications**:
- *Period ≫ τ (low rep rate)*: skip the time-stepping and run the steady-state
  BVP with signal = 0 to get the fully-recovered inversion. Exact in the limit
  of infinite inter-pulse time.
- *Period ≪ τ (high rep rate)*: the inversion barely changes between pulses. Run
  the steady-state BVP with the time-averaged signal power as the signal channel
  (the quasi-CW operating point).

#### Sub-regime B2: Pulse propagation (ns timescale)

During the pulse the propagation PDE is solved for **the signal and the pump
only**:

```
(1/v_g) ∂P_k/∂t + ∂P_k/∂z = g_k · P_k - α_bg · P_k       (k ∈ {pump, signal})
∂n₂/∂t = R_abs - R_em - n₂/τ
```

**ASE is neglected during B2.** ASE accumulates on the µs–ms inter-pulse
timescale (B1); over a 4–8 ns pulse it has no time to build up meaningfully and
cannot drain the inversion. This is the standard two-part MOPA decomposition:
B1 = pumping + ASE recovery → produces the inversion the pulse will see;
B2 = ASE-free pulse extraction → produces the amplified pulse energy and the
residual inversion. The two are iterated to periodic steady state.

**Algorithm: Lax-Wendroff finite difference** on a (z, t) grid:
```
Δz = L / N_z           (N_z = 200 spatial points)
Δt = Δz / v_g          (CFL = 1 — exactly one grid cell per time step)
N_t = pulse_window / Δt (pulse_window ≈ 3 × pulse_duration)
```

At CFL = 1 the upwind step reduces to the method of characteristics with an exact
per-cell exponential transmission `exp((g − α_bg)·Δz)` plus an upstream-source
term. At each time step, for the pump and signal channels only:
```
forward (u_k = +1):
    P_k(z_i, t+Δt) = P_k(z_i, t) - (v_g Δt / Δz) · [P_k(z_i, t) - P_k(z_{i-1}, t)]
                     + Δt · v_g · g_k(z_i) · P_k(z_i, t)

Update n₂(z_i):
    n₂(z_i, t+Δt) = n₂(z_i, t) + Δt · [R_abs(z_i, t) - R_em(z_i, t) - n₂(z_i, t)/τ]
```

**Boundary conditions during pulse**:
```
P_signal_fwd(z=0, t) = input_pulse_shape(t)    (Gaussian)
P_pump_fwd(z=0, t)   = P_pump (CW — the pump stays on through the pulse)
```

### 4.3 Handoff between modes

```
Full simulation cycle for one rep-rate period:

1. INTER-PULSE: Start from depleted n₂(z) after previous pulse.
   Run Mode B1 (or Mode A if period >> τ) for duration 1/f_rep.
   Result: recovered n₂(z) profile just before pulse arrives.

2. PULSE: Run Mode B2 with n₂(z) from step 1 as initial condition.
   Signal pulse enters at z=0, propagates, extracts energy.
   Result: amplified pulse shape, depleted n₂(z).

3. Repeat for next pulse (or detect convergence to periodic steady state):
   max |n₂_before_pulse(z, cycle N) - n₂_before_pulse(z, cycle N-1)| < cycles_tol

Default cycles_tol = 1e-3, cycles_max = 25.
```

### 4.4 The layered robust steady-state solver

`Amplifier.propagate(mode="steady")` does not call the bare shooter directly — it
calls `solve_steady_state_robust`, a three-layer cascade that catches
energy-conservation–violating fixed points (a high-gain BVP can admit multiple
self-consistent solutions; see Part II §13b for the full design, diagnostics, and
literature backing):

1. **Layer 1 — direct solve + health diagnostics.** Run Mode A, then check the
   global photon balance against the quantum-defect ceiling
   `signal_gain + ASE_out + bg_loss ≤ QD · pump_absorbed`. Within ~1% → return;
   1–5% over → return with warning; >5% → Layer 2.
2. **Layer 2 — Ren et al. 2015 homotopy continuation.** Walk from an easy
   operating point (pump-only → add signal → enable ASE source → ramp pump
   50%→75%→100%), each step warm-starting the next via the `init=` parameter,
   with an Xu et al. 2014 linear-gain-shape multistart as final fallback.
3. **Layer 3 — time-marching arbiter.** Dispatch to `solve_time_dependent(mode="full")`;
   the time-dependent IVP has a unique attractor by construction, so it cannot
   land on an energy-violating fixed point. If even that bails, return Mode A's
   runaway-clamped result with `solver_path_used="all_failed"`.

## 5. Multi-Stage Coupling

### 5.1 Forward ASE propagation between stages

After stage i, the forward ASE spectrum at z = L passes through inter-stage
components, each applying a spectral transfer function:

```
Isolator:        T(λ) = 10^(-loss_dB / 10)    for all λ (flat, ~0.5–1.0 dB)
                 Backward: T = 0 (blocks backward propagation between stages)

Bandpass filter: T(λ) = T_peak · exp(-4·ln2·((λ - λ_center)/FWHM)²)
                 Typical: λ_center = 1064 nm, FWHM = 2 nm, T_peak ≈ -0.5 dB
                 Crushes the 1030 nm ASE peak before the next gain stage.

MFA / combiner:  T(λ) = 10^(-loss_dB / 10)    flat loss

Full inter-stage transfer:
P_ASE_fwd_input_next(λ) = P_ASE_fwd_out(λ) · T_iso(λ) · T_filter(λ) · T_mfa(λ)
```

### 5.2 Backward isolation

Isolators between stages block backward ASE from stage i+1 reaching stage i, so
each stage's backward ASE is self-contained. In the simulation, **each stage is
solved independently for backward ASE**. With imperfect isolation (e.g. 30 dB),
a small fraction couples back:

```
P_ASE_bwd_input(z=L, stage i) = P_ASE_bwd_output(z=0, stage i+1) · 10^(-isolation_dB/10)
```

Negligible at 30 dB, but the model supports it.

### 5.3 End-facet reflections

```
Flat-cleaved:     R ≈ 3.5% = 0.035
Angle-cleaved 8°: R < 10⁻⁴      ← simulator default
AR-coated:        R < 10⁻³
```

BVP boundary conditions:
```
P_ASE_bwd(z=L) = R_output · P_ASE_fwd(z=L)      for each ASE bin
P_ASE_fwd(z=0) = R_input · P_ASE_bwd(z=0)        for each ASE bin
```

**Note on facet reflections**: the BVP includes the facet-reflection boundary
conditions above so the solver sees any feedback they create. Whether such
feedback drives the fiber into oscillation is a hardware concern and is **out of
scope** for this simulator — it is not diagnosed here. If the feedback (or simply
very high gain) makes the iteration diverge, the runaway guard in
`solver_steady.py` clamps the result and raises the generic `solver_failed`
flag; see Part II §13.

## 6. Data Structures

### 6.1 Spectral grid (initialize once)

```python
@dataclass
class SpectralGrid:
    lambda_min: float = 970e-9       # [m]
    lambda_max: float = 1130e-9      # [m]
    d_lambda: float = 1e-9           # [m]

    # Derived (compute in __post_init__):
    n_bins: int                       # = 160
    wavelengths: np.ndarray           # center wavelengths [m], shape (n_bins,)
    frequencies: np.ndarray           # center frequencies [Hz], shape (n_bins,)
    d_nu: np.ndarray                  # bin widths in Hz, shape (n_bins,)
    sigma_a: np.ndarray               # absorption cross-section [m²], shape (n_bins,)
    sigma_e: np.ndarray               # emission cross-section [m²], shape (n_bins,)
    overlap: np.ndarray               # overlap factor Γ(λ), shape (n_bins,)
```

`SpectralGrid.from_fiber(r_core, NA)` precomputes per-bin σ_a(λ), σ_e(λ), Γ(λ),
A_eff(λ).

### 6.2 Fiber state (evolves during simulation)

```python
@dataclass
class FiberState:
    z_grid: np.ndarray                 # spatial grid [m], shape (N_z,)
    n2: np.ndarray                     # upper-state fraction, shape (N_z,)
    P_pump_fwd: np.ndarray             # forward pump [W], shape (N_z,)
    P_pump_bwd: np.ndarray             # backward pump [W], shape (N_z,)  (if counter-pumped)
    P_signal: np.ndarray               # signal power [W], shape (N_z,)
    P_ASE_fwd: np.ndarray              # forward ASE [W], shape (N_z, n_bins)
    P_ASE_bwd: np.ndarray              # backward ASE [W], shape (N_z, n_bins)
```

### 6.3 Stage output (passed between stages)

```python
@dataclass
class StageOutput:
    signal_pulse: np.ndarray           # P_signal(t) temporal profile [W]
    t_grid: np.ndarray                 # time grid [s]
    ase_fwd_spectrum: np.ndarray       # P_ASE_fwd(λ) at fiber output [W]
    ase_bwd_spectrum: np.ndarray       # P_ASE_bwd(λ) at fiber input [W]
    n2_profile: np.ndarray             # final n₂(z)
    signal_energy_out: float           # integrated pulse energy [J]
    total_ase_fwd: float               # integrated forward ASE power [W]
    total_ase_bwd: float               # integrated backward ASE power [W]
    gain_dB: float                     # signal gain
    solver_failed: bool                # True if the solve diverged (no stable steady state)
```

In the running code these roles are filled by `AseState` / `OpticalState`
(`ase/state.py`) and `SteadyResult` (`ase/solver_steady.py`).

## 8. Algorithm: Complete Simulation Flow

```
INITIALIZATION:
1. Create SpectralGrid (wavelength bins, cross-sections, overlap factors)
2. Create fiber specifications for each stage
3. Define seed pulse (average power, rep rate, pulse duration, linewidth)
4. Define passive components with spectral transfer functions

STAGE-BY-STAGE SIMULATION (for each stage i):

    INPUT: P_signal_in, P_ASE_fwd_in(λ) (zero for stage 1, else filtered
           output of stage i-1), P_pump for this stage.

    STEP 1: INTER-PULSE STEADY STATE
    - signal = 0 (or time-averaged signal power if rep is high)
    - Solve the steady-state BVP with pump + ASE
    - Result: n₂(z), P_ASE_fwd(z,λ), P_ASE_bwd(z,λ); flag if the solve diverged

    STEP 2 (if fully time-resolved): INTER-PULSE DYNAMICS (Mode B1)
    - Time-step through the inter-pulse interval from the depleted n₂(z)

    STEP 3: PULSE PROPAGATION (Mode B2, Lax-Wendroff)
    - Propagate signal pulse + pump; n₂(z) depletes; ASE neglected on ns scale

    STEP 4: EXTRACT OUTPUT
    - Signal gain, forward ASE spectrum at z=L, backward ASE spectrum at z=0

    STEP 5: APPLY INTER-STAGE COMPONENTS
    - Multiply P_ASE_fwd(λ) by T(λ); apply signal insertion loss; feed next stage

FINAL OUTPUT: amplified pulse, total/forward ASE spectrum, per-stage gain &
ASE power, solver-health flags, signal-to-ASE ratio (OSNR).
```

## 9. Simplification Levels

Valid intermediate implementations, most to least simplified. The simulator
implements **Level 5** (and auto-dispatches to the Level-3 steady state when the
rep rate is high enough that the inversion barely changes between pulses).

- **Level 1** — steady-state spatial + single ASE band (the old scalar model).
  Can underestimate ASE by 10–100× in high-gain stages.
- **Level 2** — steady-state spatial + spectrally resolved *forward* ASE only.
  Misses ~50% of ASE (backward contributes to inversion depletion).
- **Level 3** — steady-state BVP with full bidirectional spectral ASE. The
  minimum viable accurate model; valid when rep_rate ≫ 1/τ (~1.2 kHz).
- **Level 4** — Level 3 + pulsed energy extraction (Frantz-Nodvik). Captures
  rep-rate dependence without a full time-domain solver.
- **Level 5** — full time-dependent (z, t) solver (§4.2–§4.3). Captures pulse
  shape distortion and inter-pulse ASE dynamics. Most accurate, most expensive.

## 10. Validation Benchmarks

- **10.1 Zero-signal ASE-only**: signal = 0, full pump. Forward ASE peaks near
  the Yb gain wavelength (~1030 nm); backward ASE shows additional 975 nm
  structure; total ASE grows with length then saturates as it clamps the
  inversion.
- **10.2 High-signal transparency**: signal ≫ P_sat depletes the inversion → ASE
  negligible, gain → transparency.
- **10.3 Frantz-Nodvik limit**: for a single pulse extracting from a fully
  inverted fiber (no pump, no ASE during pulse),
  `E_out = E_sat · ln(1 + G₀ · [exp(E_in/E_sat) - 1])`, where
  `G₀ = exp(∫₀ᴸ g₀(z) dz)`. The Level 5 solver reproduces this.
- **10.4 Gain clamping**: increasing pump with no signal plateaus the small-signal
  gain — ASE clamping.
- **10.5 Rep-rate dependence**: at fixed pump and pulse energy, low rep → full
  recovery, max ASE, max per-pulse energy; high rep → minimal recovery, minimal
  ASE. The simulator reproduces this monotonic relationship.
- **10.6 PyFiberAmp cross-check**: the steady-state forward/backward ASE spectra
  should agree with PyFiberAmp within a few percent for the same configuration
  (see `validation/`).

## 11. Common Implementation Pitfalls

1. **Forgetting the 975 nm region.** Starting the grid at 1000 nm misses the
   strong backward ASE near 975 nm. Always start at or below 970 nm.
2. **Using A_core for pump intensity geometry incorrectly.** The pump propagates
   in the cladding; the correct rate is `σ_a · P / (h·ν·A_clad)`, obtained in the
   implementation via `A_dope = A_core` with `Γ_pump = A_core/A_clad` (Part II §6).
3. **Not summing ASE in the rate equation.** n₂(z) depends on ALL channels —
   pump, signal, and every ASE bin in both directions. Omitting ASE overestimates
   the inversion and the gain.
4. **Sign conventions for backward channels.** Backward channels propagate in -z;
   integrate from z=L to z=0 (or negate the equation and integrate forward).
5. **Numerical instability at high gain.** If small-signal gain exceeds ~30 dB,
   forward propagation can overflow. The local-n₂ update (Part II §11), the
   runaway sentinel, and the layered robust solver address this.
6. **Pretending a diverged solve is a steady state.** If the iteration runs
   away (very high gain and/or facet feedback), clamp and raise a generic
   solver-failure flag rather than reporting the clamped numbers as physical.
7. **Δν vs Δλ confusion.** The source term uses Δν (frequency width):
   `Δν_i = c · Δλ / λ_i²`, which is not equal across bins.

# Part II — Walkthrough

This part explains, step by step, how the simulator computes ASE inside each
amplifier stage. No programming knowledge is required. It uses the equations
defined rigorously in Part I.

## 1. Why ASE Needs Its Own Solver

ASE is broadband, incoherent light generated by Yb³⁺ ions decaying spontaneously
and getting amplified as they propagate. Unlike the signal — a coherent pulse at
exactly 1064 nm — ASE:

1. **Spans a broad wavelength range** (roughly 970–1130 nm for Yb).
2. **Travels in BOTH directions** (forward, with the signal, and backward,
   toward the input).
3. **Eats inversion** — every photon of ASE is built from an excited Yb ion, so
   ASE depletes the energy that should go into the signal pulse.

A scalar model that approximates ASE as a single power number with a fixed
bandwidth at 1064 nm misses every one of these. In particular it underestimates
true ASE in high-gain stages because (a) the actual gain peak is at ~1030 nm, not
1064 nm — so the scalar model puts ASE at the wrong color; and (b) without
backward ASE, the inversion looks larger than it is, making gain look better than
it is.

The solver tracks ASE properly: 160 wavelength bins, both directions, all coupled
to the inversion equation.

---

## 2. The Spectral Grid — Slicing the ASE Spectrum

We discretize 970–1130 nm into 160 bins, each 1 nm wide (Part I §3.1). Bin `i`
is centered at `λ_i = λ_min + (i + 0.5)·Δλ`, so bin 0 is at 970.5 nm and bin 159
at 1129.5 nm. Each bin has a center frequency `ν_i = c/λ_i` and a frequency width
`Δν_i = c·Δλ/λ_i²`.

**Important:** `Δν_i` is NOT the same for every bin, because `ν = c/λ` is
nonlinear. A 1 nm bin at 970 nm covers ~318 GHz; the same 1 nm bin at 1130 nm
covers ~235 GHz. The spontaneous-emission source (§7) uses `Δν_i`, so this
matters for energy book-keeping.

The signal at 1064 nm and the pump at 976 nm are **separate channels**, not bins.

```
Total optical channels = 1 pump + 1 signal + 160 forward ASE + 160 backward ASE = 322
```

Every one of these 322 channels feeds into the inversion equation (§5). Leaving
any of them out (the easy mistake) makes ASE depletion invisible to the inversion.

---

## 3. Cross-Sections — How Strongly Yb Absorbs and Emits at Each Wavelength

Two wavelength-dependent quantities govern how Yb interacts with light:

- **σ_a(λ)** = absorption cross-section — how strongly a ground-state ion absorbs
  a photon at λ.
- **σ_e(λ)** = emission cross-section — how strongly an excited ion is stimulated
  to emit at λ.

Reference values are in Part I §3.2. Three things to notice:

1. **Both cross-sections are huge at 976 nm (the zero line)** — the pump
   wavelength, strongly absorbed.
2. **σ_e dominates σ_a, and σ_e rises 1000 → 1020 nm** — the emission shoulder
   peaks just past 1020 nm, which is why that region has the largest gain ratio
   σ_e/σ_a.
3. **Below 1000 nm, σ_a is enormous** — ASE generated near 976 nm gets
   reabsorbed almost immediately, producing the "dip" in the backward ASE
   spectrum.

The values live in `ase/data/cross_sections_yb.csv` (measured Melkumov AS table)
and are interpolated (piecewise log-linear, monotone between measured nodes) onto
each amplifier's spectral grid at startup, giving each fiber per-bin σ_a, σ_e
arrays of length 160.

---

## 4. Mode Overlap Γ(λ) — Why Per-Bin Values Matter

Even though all 160 bins propagate in the same fiber, they don't see the same
mode. A photon at 970 nm has a tighter mode than one at 1130 nm, because the
V-number depends on wavelength:

```
V(λ) = (2π / λ) · r_core · NA
```

Smaller λ → larger V → tighter mode → larger overlap with the doped core. For
each bin we compute the mode radius `w(λ)` from the Marcuse approximation, then:

```
Γ(λ)   = 1 - exp(-2 · (r_core / w(λ))²)      (overlap with the doped core)
A_eff(λ) = π · w(λ)²
```

Across the band Γ varies by ~10% and A_eff by ~16% — enough to matter for
per-bin gain. The pump is different: it propagates in the cladding, so
`Γ_pump = A_core / A_clad` (very small — that's why cladding pumping needs lots
of pump power).

---

## 5. The Inversion Equation — Summed Over All 322 Channels

At every position z, the excited fraction `n₂(z)` is set by the balance of
pumping-up and decaying-down (Part I §2.5):

```
n₂(z) = numerator(z) / [denominator(z) + 1/τ]
```

where each optical channel `k` contributes:

```
numerator term:    σ_a,k · Γ_k · P_k(z) / (h · ν_k · A_core)        ← drives n₂ UP (absorption)
denominator term:  (σ_a,k + σ_e,k) · Γ_k · P_k(z) / (h · ν_k · A_core)  ← saturation
```

Summed over all 322 channels. Constants: `h` = 6.626 × 10⁻³⁴ J·s,
`τ` = 0.83 ms (Yb upper-state lifetime), `A_core` (the doped area — see §6).

**The crucial point:** every channel — including all 160 backward ASE bins —
appears in both sums. This is what makes ASE actually deplete the inversion. Omit
the ASE bins (as the old scalar model effectively did) and n₂(z) comes out too
high, so the simulated gain is unrealistically large.

---

## 6. A Convention Note: A_clad vs A_core for the Pump

A naïve reading would set "A_k = A_clad for the pump channel." Combined with
Γ_pump = A_core/A_clad, that double-counts the geometric dilution and gives a
pump rate ~1000× too small.

The textbook formulation (Becker et al.; Snitzer & Po) uses **A_core for ALL
channels**, with Γ_pump = A_core/A_clad expressing the cladding-mode overlap. For
the pump this simplifies cleanly:

```
σ_a · Γ_pump · P / (h · ν · A_core)
  = σ_a · (A_core/A_clad) · P / (h · ν · A_core)
  = σ_a · P / (h · ν · A_clad)              ← the standard cladding-pumped result
```

Same answer, no double-counting. The simulator uses `A_dope = A_core`
everywhere; this matches the scalar `components.py` rate equation and produces
physically correct inversions. The choice is documented in
`ase/solver_steady.py:_compute_n2_local`.

---

## 7. Spontaneous Emission Source — Where ASE Comes From

Each ASE bin has a built-in source: photons spontaneously emitted by excited ions
that couple into that bin's guided mode (Part I §2.4):

```
S_k(z) = m_pol · h · ν_k · Δν_k · Γ_k · σ_e,k · N_Yb · n₂(z)
```

- `m_pol = 2` for non-PM fiber (two polarizations) or `1` for PM.
- `h · ν_k` is the energy per photon.
- `Δν_k` is the bandwidth this bin captures — wider bin → more photons.
- `Γ_k · σ_e,k · N_Yb · n₂(z)` is the per-volume spontaneous emission rate into
  the mode.

Pump and signal channels do NOT have this source — they're externally injected.
Forward and backward ASE share the same source magnitude (spontaneous emission is
isotropic).

---

## 8. The Propagation Equations — How Powers Change Along z

For each channel k with direction `u_k` (+1 forward, -1 backward), in steady
state:

```
u_k · dP_k/dz = u_k · g_k(z) · P_k(z)   (gain/absorption)
              + u_k · S_k(z)            (spontaneous emission, ASE bins only)
              - u_k · α_bg · P_k(z)     (background loss, ~0.005 dB/m)
```

with the gain coefficient

```
g_k(z) = Γ_k · N_Yb · [n₂(z) · σ_e,k - (1 - n₂(z)) · σ_a,k]
```

A channel is amplified when n₂ is above its transparency point
`n₂_transparent,k = σ_a,k / (σ_a,k + σ_e,k)`. For the signal at 1064 nm this is
~3%, so almost any inversion gives signal gain. For a 975 nm bin it's ~50%, so
you need n₂ above 50% before that bin amplifies rather than absorbs — which is
why backward ASE shows the 975 nm dip even when the fiber is well inverted.

```
Forward (u_k = +1):   dP_fwd/dz = +(g_k - α_bg) · P_fwd + S_k     (S only for ASE bins)
Backward (u_k = -1):  dP_bwd/dz = -(g_k - α_bg) · P_bwd - S_k
```

---

## 9. Boundary Conditions — Why This Is a BVP

Some powers are known at the input (z = 0), others at the output (z = L); none
fully known at both. That makes it a **boundary-value problem**.

```
At z = 0:  P_pump(0), P_signal(0), P_ASE_fwd(0, λ) known;  P_ASE_bwd(0, λ) UNKNOWN
At z = L:  P_ASE_bwd(L, λ) = R_out · P_ASE_fwd(L, λ)        (Fresnel reflection)
```

`R_out` is the output power reflectivity (≈ 3.5% flat-cleaved; < 10⁻⁴
angle-cleaved — the default; < 10⁻³ AR-coated). Forward and backward fields are
coupled (backward ASE depletes the inversion the forward fields see, and forward
ASE feeds the backward field at z = L through the reflection), so solving them
requires iteration.

---

## 10. The Iterative-Shooting Solver — One Iteration at a Time

The solver lives in `ase/solver_steady.py` (Part I §4.1):

```
Initialize fields (P_ase_bwd seeded from spontaneous emission, NOT zero — §11).

Repeat until converged:
    1. Forward sweep z=0 → z=L: RK4-step the pump, signal, and forward-ASE
       channels, recomputing n₂(z) LOCALLY from the just-updated P at each step.
    2. Apply boundary condition at z=L:  P_ase_bwd(L, λ) = R_out · P_ase_fwd(L, λ).
    3. Backward sweep z=L → z=0: RK4-step the backward-ASE channels, again with
       the local-n₂ update.
    4. Convergence: max relative change in any channel < 1e-5.

Return n₂(z), P_pump(z), P_signal(z), P_ase_fwd(z, λ), P_ase_bwd(z, λ).
```

The key trick is the local-n₂ update inside each sweep (step 1), which deserves
its own section.

---

## 11. Why Local n₂ Prevents Runaway

A naïve implementation freezes n₂(z) for the whole iteration: compute it once,
then sweep forward and backward without changing it. That fails badly in
high-gain stages.

Consider a well-inverted high-gain stage. From a "constant pump, no ASE" trial,
the inversion comes out well above transparency everywhere, giving a large gain
coefficient at the 1030 nm peak — tens of dB per meter. Over a few meters that's
a hundred-odd dB of single-pass gain, so a tiny spontaneous source becomes
astronomically amplified after one sweep, and the backward ASE explodes too.
Numerically the run blows up; physically the system would have saturated long
before.

The fix is to update n₂ at every step of the sweep using the *just-computed*
local P values. As ASE grows along z, n₂ at the new z drops to reflect the new
saturation, and the next step uses that smaller gain coefficient. The exponential
blow-up self-limits — saturation is built into the sweep, not deferred to the
next iteration. This is what lets the solver converge in a handful of iterations;
without it, even heavy underrelaxation fails on a high-gain stage.

The initial guess itself is not zero-ASE, either: the default seeds the forward
**and backward** ASE fields from spontaneous emission on a pump-only inversion
profile (`_spontaneous_emission_init`). A zero-ASE start lets a high-gain fiber's
first sweep amplify a negligible source into a runaway and settle in the
unphysical ASE-dominated basin (§13b); the spontaneous-emission floor lands the
iteration in the physical, signal-dominated basin instead.

---

## 12. Convergence and the Runaway Guard

At the end of each iteration the solver computes the largest relative change in
any channel at any z:

```
max_rel = max over k, z of  |P_k_new(z) - P_k_old(z)| / max(|P_k_new(z)|, 1e-15)
```

If `max_rel < 1e-5` (after at least 2 iterations), it declares convergence. If it
hasn't converged after 100 iterations, it returns with `converged=False` and a
note.

There's also a **runaway sentinel**: if any single bin's power exceeds 1 kW
during an iteration, the solver bails, clamps the spectrum, and sets
`solver_failed=True`. This catches configurations where no stable steady state
exists (§13).

---

## 13. Solver-Health Flag — When Steady State Doesn't Exist

Very high single-pass gain — and/or feedback from end-facet reflections — can
push the BVP into a regime where the iterative shooter has no stable steady
state to converge to: the ASE (or signal) field grows without bound from one
iteration to the next. Whether such a fiber would physically build an
oscillating mode is a **hardware question that is out of scope** for this
simulator; we do not attempt to diagnose or characterize it.

What the solver *does* do is refuse to report untrustworthy numbers. The
**runaway sentinel** above detects the divergence, clamps the offending
spectrum, and sets the generic `solver_failed=True` flag with an explanatory
note. The report then tags the stage `[SOLVER ISSUE: no stable steady state]`
and the V&V `solver_stable` check fails. `converged=False` accompanies it.

This is also why angle-cleaved facets (R ≈ 10⁻⁴) matter in practice: keeping
reflections tiny keeps feedback out of the gain path. Lengthening the fiber or
raising the pump both raise single-pass gain and can push a stage into the
no-stable-solution regime; the `solver_failed` flag tells you when the solver
could not produce a trustworthy result there.

---

## 13b. The Layered Robust Solver — Catching Non-Physical Fixed Points

The runaway sentinel only catches outright divergence. There's a subtler failure
mode it can't catch on its own: when the gain is high enough that the BVP admits
multiple self-consistent fixed points, the iterative shooter can silently land on
one that converges cleanly yet violates global energy conservation.

### The failure mode (literature: well-known, not formally named)

The Giles-Desurvire propagation system at our resolution (322 channels, RK4) is a
multi-dimensional root-finding problem with strong exponential sensitivity.
Paschotta (RP Photonics), Ren et al. (Opt. Quantum Electron. 47(7), 2199, 2015),
and the `scipy.integrate.solve_bvp` documentation all describe the phenomenon:
high-gain BVPs with bidirectional ASE have **many basins of attraction**, and
Newton-type root finders converge to whichever one the initial guess lies in.
Local rate equations can be satisfied at every z to machine precision while the
*global* energy budget fails.

The global constraint is the **quantum-defect (QD) ceiling**: pump photons carry
more energy than signal photons, so

```
signal_gain_W ≤ (ν_signal / ν_pump) · pump_absorbed_W ≡ QD · pump_absorbed_W,   QD ≤ 1.
```

The difference `h·ν_pump − h·ν_signal` per transition is quantum-defect heat
deposited in the lattice. The rate equation enforces per-ion photon balance
locally; it does **not** automatically enforce the global energy budget. In
healthy regimes the BVP has a unique fixed point and the iteration finds it; near
the high-gain edge it can converge to an ASE-dominated basin where the reported
signal output exceeds the QD ceiling by several times — mathematically
self-consistent, physically impossible. (The spontaneous-emission seed of §11 is
the first line of defense; the layers below are the safety net for what it
doesn't rescue.)

### The remedy — three layers + three diagnostics

The robust entry point `ase/solver_time.py::solve_steady_state_robust` wraps the
raw Mode A solver in three cascading layers (also summarized in Part I §4.4):

**Layer 1: direct solve + health classification.** Compute
`signal_gain + ASE_out + bg_loss` and compare to `QD · pump_absorbed`. Within 1%
of (or under) the ceiling → return. 1–5% over → return with a warning. >5% →
Layer 2. The 1%/5% thresholds are **engineering judgement** (no published
canonical value): ~20× the RK4-truncation + iteration noise floor (~10⁻³) and
well above the worst observed healthy regime (a fully saturated power stage runs
just under the ceiling).

**Layer 2: Ren et al. 2015 homotopy continuation.** Walk from a known-easy
operating point to the target, each step warm-starting the next via the `init=`
parameter:

1. Pump-only solve (`signal=0`, `m_pol=0` to disable the ASE source). Unique
   fixed point.
2. Add signal, keep ASE source off. Beer-Lambert-like — still unique.
3. Enable ASE source. **Critical step**: the iteration starts in the
   signal-clamped basin instead of the ASE-dominated one.
4. Parameter continuation: scale pump 50% → 75% → 100% of target, each warm-started.
5. Final fallback: Xu et al. 2014 linear-gain-shape multistart with three slope
   efficiencies (0.3, 0.5, 0.7).

References: Opt. Quantum Electron. 47(7), 2199 (2015),
DOI 10.1007/s11082-014-0096-8; Optik 2014 pii S1068520014000546.

**Layer 3: time-marching B1+B2 arbiter.** If Layer 2 still violates, the system
likely has no physical CW steady state. Dispatch to
`solve_time_dependent(mode="full")` (B1 rate-equation time-stepping + B2
Lax-Wendroff pulse PDE). The time-dependent IVP has a **unique attractor** —
landing on an energy-violating fixed point is impossible by construction. This is
the pattern PyFiberAmp uses with `DynamicSimulation(stop_at_steady_state=True)`.
If B1+B2 itself bails (the runaway/blowup guard fires), no usable steady state
exists at all; the wrapper returns Mode A's runaway-clamped result with
`solver_path_used="all_failed"`.

> **Circular-dependency note.** Layer 3 calls `solve_time_dependent(mode="full")`,
> which calls `_periodic_steady_state`, whose warmup must NOT call the full
> robust wrapper (that would recurse into Layer 3). The warmup therefore calls
> `solve_steady_state_homotopy` directly — homotopy alone never invokes the
> time-dependent solver. A separate NaN-guard inside `_periodic_steady_state`
> remains as a defensive backstop for configurations with no stable steady state.

### The three diagnostics

In `ase/solver_health.py`:

| Diagnostic | Formula | Threshold(s) | Reference |
|---|---|---|---|
| `energy_residual_ratio` | `(consumed - delivered) / delivered`, with `delivered = QD · pump_absorbed`, `consumed = signal_gain + ASE_out + bg_loss` | `> 0.01` warn, `> 0.05` violate | Ren et al. 2015 §2 (photon balance); engineering judgement on thresholds |
| `ase_conversion_fraction` (η_ASE) | `ASE_out_total / pump_absorbed` | `> 0.1` "mixed", `> 0.3` "sfs" | Wang & Clarkson 2006 (110 W Yb SFS); Dong 2025 operational thresholds |
| `small_signal_g0L` | `Γ_signal · N · (n2_asymp·σ_e,sig − (1−n2_asymp)·σ_a,sig) · L` | `> 30` flag "high_gain" | Furuse et al. used `g(0)·l ≈ 3` for Yb:YAG thin-disk; the fibre threshold is ~10× higher because the doped core covers only a small fraction of the mode |

### What the user sees

`Amplifier.info` gains seven keys after every steady-state solve:

```
"energy_residual_ratio":   float,
"ase_conversion_fraction": float,    # η_ASE
"small_signal_g0L":        float,
"energy_status":           "ok" | "warning" | "violation",
"regime":                  "amplifier" | "mixed" | "sfs" | "high_gain",
"solver_path_used":        "direct" | "homotopy" | "time_marching_arbiter" | "all_failed",
"homotopy_steps_used":     int,
```

The text report adds a compact `[E:.. R:.. path=..]` decoration on the gain line
whenever the solve is **not** in the clean "direct + healthy + amplifier" case
(e.g. `[E:WARNING ×1.01  R:AMPLIFIER  path=homotopy(4)]` or
`[E:VIOLATION ×5.30  R:SFS  path=all_failed]`).

### When you see `[E:VIOLATION]`

The simulator is telling you: (1) the BVP solved to tolerance — the numbers are a
*self-consistent* local solution; (2) none of direct → homotopy → time-marching
found a physical solution respecting the QD ceiling; (3) in a real lab this
configuration would not act like a clean amplifier — it would become an
ASE/superfluorescent-dominated source; (4) the action is to change the design
(shorten the fibre, reduce pump, increase seed) until the decoration goes away.

### Cost

In a healthy regime, Layer 1 passes on the first attempt — the wrapper is a thin
pass-through with one extra diagnostics call (~3% overhead). In marginal regimes,
Layer 2's continuation adds a few hundred ms. If Layer 3 fires, time-marching
adds seconds, but only for configurations on the edge of physicality.

## 14. Bandpass Filters Between Stages

The default system places a Gaussian bandpass filter (modeling the FBG) just
after each post-amplifier isolator:

```
Center wavelength:  1064 nm
FWHM:               2 nm
Peak transmission:  ~ -0.5 dB insertion loss
```

The transfer function for ASE is:

```
T(λ) = 10^(-peak_loss_dB / 10) · exp(-4 · ln(2) · ((λ - 1064 nm) / FWHM)²)
```

At 1064 nm the filter passes the peak (just the insertion loss); 34 nm off-band
at 1030 nm the ideal Gaussian is effectively zero (real FBGs floor at ~50 dB
rejection). Either way the 1030 nm gain peak is crushed before the next stage, so
downstream amplifiers only re-amplify the small fraction of ASE near the signal
wavelength — where the gain is already clamped by signal saturation, keeping the
noise figure low. The filter applies `T(λ)` to the ASE spectrum and a flat
insertion loss to the coherent signal (which sits exactly at 1064 nm, so it sees
only the peak loss).

## 15. Mode A vs Mode B — When to Use Each

The amplifier supports two solver modes via the `mode=` argument to
`Amplifier.propagate()`:

**Mode A — `"steady"`.** Quasi-CW: solve the steady-state BVP using the
time-averaged signal power. Exact when the rep-rate period is much shorter than
the upper-state lifetime.

**Mode B — `"time-dependent"` (the default).** Level 5 — the two-part MOPA
decomposition:
- **B1** (inter-pulse recovery) — adaptive rate-equation time-stepping; pumps up
  the inversion while ASE grows. **All ASE accounting happens here.**
- **B2** (pulse propagation) — Lax-Wendroff (z, t) at CFL = 1, propagating the
  **signal pulse and pump only** (ASE neglected on the ns scale). Captures
  pulse-shape distortion and gain saturation as the pulse extracts the inversion
  B1 prepared.

Auto-dispatch picks the branch by rep rate:
- *High rep* (period < 0.1·τ): delegates to Mode A — exact when n₂ barely changes
  between pulses. Same numbers, same wall time.
- *Otherwise*: runs the B1 + B2 cycle, warm-started from a Mode A solve, iterating
  until pre-pulse n₂(z) is stable (`cycles_tol = 1e-3`, `cycles_max = 25`).

| Rep rate | Period | Period / τ | Mode B branch |
| -------- | ------ | ---------- | ------------- |
| 100 kHz | 10 µs | 0.012 | high_rep_quasi_cw (= Mode A) |
| 10 kHz | 100 µs | 0.12 | periodic (B1+B2) |
| 1 kHz | 1 ms | 1.2 | periodic |
| 100 Hz | 10 ms | 12 | periodic |
| 10 Hz | 100 ms | 119 | periodic |

To **force** the B1+B2 path at any rep rate (for pulse-shape distortion studies),
pass `--force-b2` to `simulate.py` or `mode="full"` to `Amplifier.propagate()`.
With `--plots`, this also renders `pulse_shape.png` (temporal profile at z=0 vs
z=L per amplifier). The Level 5 benchmarks are covered in
`ase/tests/test_time_solver.py`.

## 16. Frantz-Nodvik Extraction — How Mode B Handles the Pulse

In Mode B's low-rep / periodic branches, the pulse extracts energy slice-by-slice
using the Frantz-Nodvik formula. For each slice [z[i], z[i+1]]:

```
E_sat   = h · ν_signal · A_eff_signal / (σ_e_signal · Γ_signal)
g0(i)   = Γ_signal · N_Yb · (n₂(z[i])·σ_e - (1-n₂(z[i]))·σ_a)
G0(i)   = exp(g0(i) · dz)
E_out   = E_sat · ln(1 + G0(i) · (exp(E_in / E_sat) - 1))
```

The extracted energy is `E_out - E_in`; the slice's inversion drops by
`Δn₂(i) = (E_out - E_in) / (h · ν_signal · A_core · dz · N_Yb)`. Then
`E_in_next = E_out` and the process repeats to z = L. At low rep rate the
inversion has time to fully recover between pulses, so each pulse can extract a
large fraction of the stored energy — far more than the seed pulse energy. The
B2 Lax-Wendroff path additionally captures pulse-shape distortion that the
per-slice extraction does not.

## 17. What Changed vs. the Old Scalar Model — At a Glance

| Aspect | Old (scalar) | New (BVP) |
| ------ | ------------ | --------- |
| ASE representation | Single number `P_ASE` | 160 wavelength bins (forward) + 160 (backward) |
| ASE wavelength assumption | Always at 1064 nm | Computed across 970–1130 nm |
| ASE bandwidth | Hardcoded ~5 THz | Per-bin Δν, bin-width-aware |
| Backward ASE | Not modeled | Solved as part of the BVP |
| Inversion equation | Sums pump and signal only | Sums all 322 channels |
| Solver-health flag | None | Diverged solves clamped and flagged |
| Inter-stage filtering | Flat dB loss only | Wavelength-dependent transfer function |
| Cross-section data | A few hardcoded scalars | Measured CSV interpolated to fiber grid |
| Solver | Closed-form, single forward sweep | Iterative shooting with local-n₂ update |

The architectural addition is the `ase/` package; `components.py` and
`simulate.py` keep the same shape but now exchange `OpticalState` (signal + ASE
state) instead of bare `Signal` objects. The new model is more pessimistic about
ASE in high-gain stages, which is a *correctness* improvement — that ASE was
always there; the scalar model just hid it.

## 19. Summary — The Equation Set in One Place

Per-bin gain coefficient:
```
g_k(z) = Γ_k · N_Yb · [n₂(z) · σ_e,k - (1 - n₂(z)) · σ_a,k]
```

Per-bin spontaneous emission source (ASE bins only):
```
S_k(z) = m_pol · h · ν_k · Δν_k · Γ_k · σ_e,k · N_Yb · n₂(z)
```

Propagation:
```
dP_fwd,k/dz = +(g_k - α_bg) · P_fwd,k + S_k
dP_bwd,k/dz = -(g_k - α_bg) · P_bwd,k - S_k
```

Boundary conditions:
```
At z = 0:  P_pump_fwd, P_signal_fwd, P_ASE_fwd known
At z = L:  P_ASE_bwd(λ) = R_out · P_ASE_fwd(λ)
```

Steady-state inversion summed over all 322 channels:
```
n₂(z) = [Σ_k σ_a,k · Γ_k · P_k(z) / (h · ν_k · A_core)]
       / [Σ_k (σ_a,k + σ_e,k) · Γ_k · P_k(z) / (h · ν_k · A_core) + 1/τ]
```

Solver-health flag (no dedicated equation): if the iteration diverges — any ASE
bin exceeds the 1 kW runaway sentinel — the result is clamped and
`solver_failed=True` is set, meaning no stable steady state was found.

That's the complete equation set. Everything else in `ase/` is bookkeeping —
discretization, iteration, convergence checks, plotting.
